## 1. 背景与目标

### 1.1 背景
- 公司内部有大量开发任务，但开发资源**有限且分散**在各个部门
- 已有**虚拟货币制度**（≈ 真实奖金，关系员工收益）
- 缺乏市场化的资源调配机制，导致部分任务长期无法推进

### 1.2 目标
搭建一个**内部任务交易撮合平台**，用市场化的方式让"谁有活"和"谁有空"自然对接。

### 1.3 MVP 范围声明

| 类别 | MVP 做 | 留到二期 |
|---|---|---|
| 任务流转 | 发布、投标、选标、指名、完成确认 | — |
| 资金 | ❌ 不做（走内部系统） | — |
| 信用 | — | ✅ 信用评分 |
| 匹配 | — | ✅ 任务分类 + 技能匹配 + 推送 |
| 定价 | — | ✅ 历史成交均价 |
| 仲裁 | — | ✅ 平台介入机制 |
| 用户体系 | ✅ 对接外部接口 | — |
| 附件 | ✅ 文件上传 | — |
| 老板视图 | ✅ 甘特图 | — |
| 个人主页 | ✅ 基础信息聚合 | 高级功能 |

## 2. 角色与人员体系

### 2.1 系统内角色

| 角色 | 描述 | 系统内权限 |
|---|---|---|
| **任务发布者** | 任何员工 | 浏览、发布、投标、确认完成、取消 |
| **接单开发者** | 任何员工 | 浏览、投标、协商、交付 |
| **老板** | 拥有全局视图权限 | 看所有任务的甘特图视图 |

> MVP 不区分"管理员"角色。付款账号的管理在**公司内部系统**进行，TaskHub 只引用账号 ID。

### 2.2 人员信息来源（关键决策）

**TaskHub 不维护本地用户表**，人员信息全部从**外部人员信息接口**获取：

- **不存**：用户姓名、工号、部门、邮箱等基础信息
- **只引用**：通过外部用户 ID 关联
- **缓存策略**：在 TaskHub 内部做短期缓存（建议 1 小时），减少对外部接口的调用
- **接口约定**（待补）：见 §13 遗留问题 L6

### 2.3 权限模型要点

- 权限与外部接口中的"部门 / 角色 / 职级"绑定
- MVP 阶段简化为：**所有已登录用户权限对等**，老板视图单独入口
- 二期可引入：部门隔离、职级限制等

## 3. 核心概念

| 概念 | 描述 |
|---|---|
| **任务（Task）** | 一笔开发工作的最小单位，从发布到完结有明确状态流转 |
| **投标（Bid）** | 开发者对应任务的报价响应 |
| **协作组（Collaboration Group）** | 多人协作投标时的一组开发者 |
| **付款账号（Payment Account）** | 标记任务的资金来源，平台只引用 ID |
| **附件（Attachment）** | 任务相关的文件（PRD、UI 稿、接口文档等） |
| **状态变更记录（Change Log）** | 任务进行中所有状态/字段变更的审计日志 |

## 4. 任务状态机

```
   ┌────────┐
   │ DRAFT  │  (草稿，发布者编辑中)
   └───┬────┘
       │ 发布
       ▼
   ┌────────┐
   │  OPEN  │  (招标中)
   └───┬────┘
       │ 超时无投标              │ 选标
       ▼                        ▼
   ┌────────┐              ┌──────────┐
   │ FAILED │              │ ASSIGNED │  (进行中)
   └───┬────┘              └────┬─────┘
       │ 复制新建                │ 发布者确认完成
       │                        ▼
       │                   ┌──────────┐
       └──────────────────►│COMPLETED │
                           └──────────┘

   DRAFT / OPEN / ASSIGNED → CANCELLED  (取消)
```

### 4.1 状态定义

| 状态 | 含义 | 终态？ |
|---|---|---|
| DRAFT | 草稿，未对外可见 | ❌ |
| OPEN | 招标中，对全员可见 | ❌ |
| ASSIGNED | 已选标，进行中 | ❌ |
| COMPLETED | 已完成 | ✅ |
| FAILED | 流标 | ❌（可复制新建） |
| CANCELLED | 已取消 | ✅ |

### 4.2 状态转换规则

| 起始状态 | 操作 | 结果状态 | 触发者 |
|---|---|---|---|
| DRAFT | 发布 | OPEN | 发布者 |
| DRAFT | 取消编辑 | CANCELLED | 发布者 |
| OPEN | 选标 | ASSIGNED | 发布者 |
| OPEN | 流标（超时无投标） | FAILED | 系统/定时 |
| OPEN | 撤回 | CANCELLED | 发布者 |
| ASSIGNED | 确认完成 | COMPLETED | 发布者 |
| ASSIGNED | 双方同意取消 | CANCELLED | 发布者+中标者 |
| FAILED | 复制新建 | DRAFT（新建一条） | 发布者 |
| * | 字段变更 | 状态不变，生成 Change Log | 双方 |

## 5. 核心流程

### 5.1 流程 A：任务发布（招标模式）

1. 发布者填写任务信息：标题、描述、**付款账号**、预算金额、期望交期、招标截止时间
2. **发布者线下与付款账号管理员确认金额**（平台不拦截）
3. 点击"发布" → 状态变 `OPEN`，对全员可见

> 金额一旦发布后变更，需发布者**线下再次找管理员确认**。

### 5.2 流程 B：投标

#### 单人投标
- 填写：投标金额、交期、可选方案描述
- 同一任务对同一开发者**只能投标一次**（可撤回后重投）

#### 多人协作投标
- 主对接人 A 填写：投标金额、交期、方案
- 在"协作成员"中**从人员接口下拉选择**其他成员
- 协作成员只需要被加入，不需要二次确认（MVP 简化）

### 5.3 流程 C：选标

1. 发布者浏览所有投标（**互相不可见**，A 不知道 B 的报价）
2. 选中一个投标
3. **可调整**：最终金额、最终交期、任务明细
4. 系统自动通知中标者，**其他投标者状态变为 LOST**（无主动通知）
5. 任务状态变 `ASSIGNED`

### 5.4 流程 D：直接指名

1. 发布者填写任务信息 + **被指名人**（从人员接口选）+ 金额
2. 状态直接变 `ASSIGNED`，**无需被指名人接受**（MVP 简化）
3. 被指名人收到通知

> 未来可加：被指名人拒绝 → 回到 OPEN。

### 5.5 流程 E：老板加塞 / 协商延期

> 老板有了新紧急任务，要求开发者放下手头的事先做新的。

1. 开发者接了新任务（`ASSIGNED`）
2. 开发者对**现有进行中的任务**发起"协商延期"
3. 线下与现有任务发布者沟通
4. 双方达成一致后，发布者点击"同意延期"
5. 记录新的交期 + 原因 + 双方确认
6. 任务状态保持 `ASSIGNED`，变更记录入 Change Log

> **不强制**：协商失败时，由双方自行找上司处理，平台不介入。

### 5.6 流程 F：任务变更 / 取消

#### 变更（进行中）
- 可变更字段：需求描述、最终金额、最终交期
- **必须双方确认**才生效
- 所有变更入 Change Log

#### 取消（任意状态）
- DRAFT / OPEN：发布者单方取消
- ASSIGNED：**必须双方同意**
- 取消后状态 `CANCELLED`
- **MVP 不做已做工时的折算**（不记录工时）

### 5.7 流程 G：完成确认

1. 开发者交付
2. 线下与发布者沟通
3. 发布者在系统上点击"确认完成"
4. 状态变 `COMPLETED`，变更记录入档

> MVP 不做超时自动机制。发布者不点完成 = 任务永远在 `ASSIGNED`，**双方自行找上司**处理。

### 5.8 流程 H：流标与重发

1. OPEN 状态超过招标截止时间，**无投标** → 状态变 `FAILED`
2. 发布者可点击"复制新建"
3. 复制原任务的**模板字段**（标题、描述、预算范围等），**不复制历史投标**
4. 新建任务进入 `DRAFT` 状态

## 9. 老板视图（甘特图）

### 9.1 入口
- 老板账号登录后，主导航有"老板视图"入口
- 普通用户**不可见**此入口

### 9.2 视图规格
- **图表类型**：甘特图（Gantt）
- **X 轴**：时间
- **Y 轴**：任务列表（默认按创建时间倒序）
- **任务条**：从任务创建到最终交期的时间跨度
- **颜色按状态**：
  - OPEN：蓝色（招标中）
  - ASSIGNED：橙色（进行中）
  - COMPLETED：绿色（已完成）
  - FAILED：灰色（流标）
  - CANCELLED：浅灰 + 删除线（已取消）

### 9.3 交互
- 点击任务条 → 跳转到任务详情
- 顶部筛选：按部门、按发布者、按状态
- 时间范围：默认最近 90 天，可选本周 / 本月 / 本季

### 9.4 数据范围
- 默认看**全部任务**
- 可按付款账号（部门）筛选

## 11. MVP 范围（最终版）
