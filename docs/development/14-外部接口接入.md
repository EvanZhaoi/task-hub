# 14-外部接口接入

## 本章目标

本章从“你还没有创建付款账号接口和本据点人员接口相关文件”的状态开始，完整完成两类外部大列表接口接入：

1. 付款账号列表接口：用于发布任务时在 Select 中选择交易/付款账号，保存任务时只保存选中的账号 ID 和账号历史快照。
2. 本据点全员人员列表接口：用于登录时给 Session 追加本据点人员信息，未来用于开发者、协作者选择器。

SSO 当前登录人接口已经在第 09 章完成，本章不重写 SSO 登录逻辑，只在登录成功后补充“本据点人员信息增强”。

## 学习目标

完成本章后，你应该理解：

- Laravel 项目中为什么把外部接口放在 `app/Integrations`。
- 为什么 Controller 不直接写 `Http::get()`。
- 为什么付款账号和人员列表要走 Redis 缓存。
- 为什么定时刷新失败或返回空数据时不能覆盖旧缓存。
- 为什么 Session 顶层保留总部 SSO 信息，而把本据点人员信息放在额外 `siteUser` 数组中。
- 如何用 Artisan 命令刷新外部接口缓存。
- 如何用 `Http::fake()` 测试外部接口，不真实访问公司系统。

## 第 1 步：补充环境变量

文件：

```text
.env.example
```

新增或确认以下内容：

```env
# 普通 Laravel 缓存仍使用 file。
# 这样 Session、Queue 和普通缓存不会被 Redis 绑定。
CACHE_STORE=file

# Redis 连接配置。
# 当前 Redis 只用于缓存外部大列表，不用于 Session 和 Queue。
REDIS_CLIENT=phpredis
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379
REDIS_DB=0
REDIS_CACHE_DB=1

# 外部大列表缓存统一使用 Redis。
EXTERNAL_DIRECTORY_CACHE_STORE=redis

# 外部大列表缓存 TTL，单位秒。86400 秒 = 1 天。
EXTERNAL_DIRECTORY_CACHE_TTL=86400

# 付款账号接口。
PAYMENT_ACCOUNT_BASE_URL=
PAYMENT_ACCOUNT_LIST_PATH=
PAYMENT_ACCOUNT_DETAIL_PATH=
PAYMENT_ACCOUNT_METHOD=GET
PAYMENT_ACCOUNT_TIMEOUT=3
PAYMENT_ACCOUNT_VERIFY_SSL=false
PAYMENT_ACCOUNT_CACHE_KEY=taskhub:payment_accounts

# 本据点全员人员接口。
PERSONNEL_BASE_URL=
PERSONNEL_LIST_PATH=
PERSONNEL_METHOD=GET
PERSONNEL_TIMEOUT=3
PERSONNEL_VERIFY_SSL=false
PERSONNEL_CACHE_KEY=taskhub:personnel_users
```

为什么这样做：

- `CACHE_STORE=file` 保持 Laravel 普通缓存简单。
- `EXTERNAL_DIRECTORY_CACHE_STORE=redis` 只让外部大列表使用 Redis。
- 付款账号和人员列表是外部主数据，不进 MySQL。
- Redis 中缓存的是选择器数据，不是业务事实数据。

修改 `.env` 后执行：



## 第 4 步：创建付款账号值对象

文件：

```text
app/Integrations/Payment/PaymentAccount.php
```

职责：

- 把外部账号 JSON 转成稳定 PHP 对象。
- 生成保存到 `task.payment_account_snapshot` 的快照。
- 生成写入 Redis 的普通数组。

关键代码结构：

```php
<?php

namespace App\Integrations\Payment;

/**
 * 外部付款账号值对象。
 *
 * TaskHub 不维护付款账号主数据。
 * 发布任务时只把外部接口返回的关键信息保存为历史快照。
 */
final readonly class PaymentAccount
{
    public function __construct(
        private string $accountId,
        private ?string $accountName = null,
        private ?string $departmentId = null,
        private ?string $departmentName = null,
        private array $raw = [],
    ) {}

    public static function fromPayload(string $requestedAccountId, array $payload): self
    {
        // 兼容直接返回账号字段，或包在 account/data 字段下。
        $account = match (true) {
            isset($payload['account']) && is_array($payload['account']) => $payload['account'],
            isset($payload['data']) && is_array($payload['data']) => $payload['data'],
            default => $payload,
        };

        // 兼容 camelCase、snake_case 和 id。
        $accountId = $account['accountId'] ?? $account['account_id'] ?? $account['id'] ?? $requestedAccountId;

        if (! is_string($accountId) || $accountId === '') {
            throw new PaymentAccountException('Payment account response does not contain account id.');
        }

        return new self(
            accountId: $accountId,
            accountName: self::nullableString($account['accountName'] ?? $account['account_name'] ?? $account['name'] ?? null),
            departmentId: self::nullableString($account['departmentId'] ?? $account['department_id'] ?? null),
            departmentName: self::nullableString($account['departmentName'] ?? $account['department_name'] ?? null),
            raw: $payload,
        );
    }

    public function toSnapshot(): array
    {
        // 快照只保存业务展示需要的稳定字段，不把外部接口原始响应整体塞进 task。
        return array_filter([
            'accountId' => $this->accountId,
            'accountName' => $this->accountName,
            'departmentId' => $this->departmentId,
            'departmentName' => $this->departmentName,
        ], fn (mixed $value): bool => $value !== null && $value !== '');
    }

    public function toCachePayload(): array
    {
        // Redis 中只缓存普通数组，不缓存 PHP 对象，避免类结构变化导致反序列化问题。
        return [
            'accountId' => $this->accountId,
            'accountName' => $this->accountName,
            'departmentId' => $this->departmentId,
            'departmentName' => $this->departmentName,
        ];
    }
}
```

说明：

- 文档只展示关键结构，实际文件还包含 getter、列表解析和空字符串处理方法。
- 你跟着写时，可以直接参考当前仓库的完整文件内容。

## 第 5 步：创建付款账号 Client

文件：

```text
app/Integrations/Payment/PaymentAccountClient.php
```

职责：

- `fetchAll()`：运行时读取付款账号列表，优先读 Redis。
- `refreshCache()`：定时任务刷新 Redis。
- `fetchById()`：发布任务保存时按账号 ID 取可信账号。

核心流程：

```text
fetchAll()
↓
读 Redis
↓
命中：返回缓存
未命中：请求外部 list_path
↓
解析成 PaymentAccount[]
↓
非空则写 Redis
```

刷新规则：

```text
refreshCache()
↓
请求外部接口
↓
返回非空：覆盖 Redis
返回空：不覆盖旧缓存
请求失败：不覆盖旧缓存
```

关键代码：

```php
public function refreshCache(): int
{
    // 定时任务调用该方法刷新 Redis。
    // 如果外部接口失败或返回空列表，不覆盖旧缓存。
    $accounts = $this->fetchAllFromRemote();

    if ($accounts === []) {
        return 0;
    }

    $this->putCacheIfNotEmpty($accounts);

    return count($accounts);
}
```

Redis 写入：

```php
private function putCacheIfNotEmpty(array $accounts): void
{
    if ($accounts === []) {
        return;
    }

    Cache::store((string) config('payment_account.cache_store', 'redis'))
        ->put(
            (string) config('payment_account.cache_key', 'taskhub:payment_accounts'),
            array_map(fn (PaymentAccount $account): array => $account->toCachePayload(), $accounts),
            (int) config('payment_account.cache_ttl', 86400),
        );
}
```

注意：

- Redis 中不存 PHP 对象，只存数组。
- 写缓存失败只记日志，不影响已经拿到的业务结果。
- 付款账号列表为空时，不更新缓存。

工号规整：

```php
public static function normalizeEmployeeNo(mixed $value): ?string
{
    if (! is_string($value) || $value === '') {
        return null;
    }

    // 只有纯数字工号去掉前导 0。
    // 例如 00010001 -> 10001。
    // 例如 E001 不处理，避免误改真实工号。
    $normalized = ctype_digit($value) ? ltrim($value, '0') : $value;

    return $normalized === '' ? '0' : $normalized;
}
```

## 第 8 步：创建人员 Client

文件：

```text

```

职责：

- `fetchAll()`：优先从 Redis 读取人员列表。
- `refreshCache()`：定时刷新 Redis。

关键方法：

```php
public function findByEmployeeNo(string $employeeNo): ?PersonnelUser
{
    $normalizedEmployeeNo = PersonnelUser::normalizeEmployeeNo($employeeNo);

    if ($normalizedEmployeeNo === null) {
        return null;
    }

    foreach ($this->fetchAll() as $user) {
        if ($user->employeeNo() === $normalizedEmployeeNo) {
            return $user;
        }
    }

    return null;
}
```

说明：

- 如果 Redis 有数据，`fetchAll()` 不会访问外部接口。
- 如果 Redis 没数据，会尝试访问外部接口并写缓存。
- 定时任务刷新失败时，不清空旧缓存。

```php
use App\Integrations\Payment\PaymentAccountClient;
use App\Integrations\Payment\PaymentAccountException;
use App\Integrations\Personnel\PersonnelClient;

use Illuminate\Support\Facades\Schedule;
```

增加命令：

```php
Artisan::command('taskhub:sync-external-directories', function (): int {
    // 外部大列表统一由定时任务刷新到 Redis。
    // 运行时页面和登录流程优先读缓存，避免每次请求都慢查询外部接口。
    $paymentAccounts = app(PaymentAccountClient::class);
    $personnel = app(PersonnelClient::class);

    try {
        $paymentAccountCount = $paymentAccounts->refreshCache();
        $this->info("Payment account cache refreshed: {$paymentAccountCount} item(s).");
    } catch (PaymentAccountException $exception) {
        // 刷新失败时不清空旧缓存。
        $this->warn('Payment account cache was not refreshed: '.$exception->getMessage());
    }

    try {
        $personnelCount = $personnel->refreshCache();
        $this->info("Personnel cache refreshed: {$personnelCount} item(s).");
    } catch (PersonnelException $exception) {
        // 刷新失败时不清空旧缓存。
        $this->warn('Personnel cache was not refreshed: '.$exception->getMessage());
    }

    return 0;
})->purpose('Refresh external payment account and personnel directories into cache');
```

增加调度：

```php
// 每 30 分钟刷新一次外部大列表。
// 服务器上需要配置 cron 执行 php artisan schedule:run。
Schedule::command('taskhub:sync-external-directories')
    ->everyThirtyMinutes()
    ->withoutOverlapping();
```


```php
private function paymentAccountOptions(PaymentAccountClient $paymentAccounts): array
{
    try {
        return array_map(
            fn (PaymentAccount $account): array => [
                // value 是发布任务表单提交给后端的账号 ID。
                'value' => $account->accountId(),
                // label 是页面展示文案。
                'label' => $account->accountName() === null
                    ? $account->accountId()
                    : "{$account->accountName()}（{$account->accountId()}）",
                'accountName' => $account->accountName(),
                'departmentName' => $account->departmentName(),
            ],
            $paymentAccounts->fetchAll(),
        );
    } catch (PaymentAccountException $exception) {
        // 账号列表不可用时，任务大厅仍可打开，前端展示不可选择。
        return [];
    }
}
```

写 Session 前：

查询本据点人员：

```php
private function siteUserFromPersonnelList(SsoUser $ssoUser, PersonnelClient $personnelClient): ?PersonnelUser
{
    try {
        $personnelUser = $personnelClient->findByEmployeeNo($ssoUser->employeeNo());
    } catch (PersonnelException $exception) {
        // 人员列表只是增强信息，失败时不阻止登录。
        Log::warning('Unable to enrich SSO user from personnel list.', [
            'employeeNo' => $ssoUser->employeeNo(),
            'message' => $exception->getMessage(),
        ]);

        return null;
    }

    return $personnelUser;
}
```

## 常见错误

| 错误 | 原因 | 处理方式 |
|---|---|---|
| `redis` 扩展不存在 | PHP 未安装 phpredis | 安装并启用 PHP Redis 扩展 |
| Redis 连接失败 | Redis 服务没启动或地址不对 | 检查 `REDIS_HOST`、`REDIS_PORT` |
| 付款账号 Select 为空 | 缓存没数据且外部接口未配置 | 执行同步命令并检查付款账号配置 |
| 登录后没有 `siteUser` | 该用户不在本据点人员列表，或人员列表缓存没数据 | 检查人员列表接口和工号格式 |
| 空数据覆盖了缓存 | 不应该发生 | 检查 `refreshCache()` 是否只在非空时写缓存 |
| 修改 `.env` 不生效 | Laravel 配置缓存未清理 | 执行 `php artisan config:clear` |

## 本章总结

本章完成的是外部大列表接入基础：

```text
付款账号列表
本据点人员列表
↓
定时刷新
↓
Redis 缓存
↓
运行时优先读缓存
↓
空数据或失败不覆盖旧缓存
```

关键原则：

- SSO 总部人员信息是认证来源，不覆盖。
- 本据点人员信息只追加到 `siteUser`。
- 付款账号名称不能由前端提交。
- 外部大列表不落 MySQL。
- Redis 只缓存外部大列表，不做 Session 和 Queue。
