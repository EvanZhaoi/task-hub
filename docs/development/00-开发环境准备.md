# 00-开发环境准备

## 1. 本章目标

确认本机具备运行 TaskHub Phase 1 的基础环境：PHP 8.5.8、Composer、Node.js、npm、MySQL 8、Redis，以及 Laravel 需要的 PHP 扩展。

## 2. 前置条件

- 可以访问命令行。
- 已安装 Git。
- 可以连接本地或远程 MySQL 8。
- 可以连接本地或远程 Redis。Redis 当前只用于缓存外部付款账号和人员大列表。

## 3. 本章最终效果

执行版本检查命令后，可以确认本机满足 TaskHub 的开发要求。

## 4. 实际执行命令

```bash
php -v
composer --version
node -v
npm -v
mysql --version
redis-server --version
php -m | grep -E "pdo_mysql|redis|mbstring|openssl|tokenizer|ctype|json|fileinfo"
```

Windows PowerShell：

```powershell
php -v
composer --version
node -v
npm -v
mysql --version
redis-server --version
php -m | Select-String "pdo_mysql|redis|mbstring|openssl|tokenizer|ctype|json|fileinfo"
```

## 5. 命令执行目录

这些命令可以在任意目录执行。

## 6. 命令作用

- `php -v`：确认 PHP 版本，TaskHub 使用 PHP 8.5.8。
- `composer --version`：确认 PHP 包管理器可用。
- `node -v` / `npm -v`：确认前端构建工具可用。
- `mysql --version`：确认 MySQL CLI 可用。
- `redis-server --version`：确认 Redis 服务端已安装。
- `php -m`：确认 PHP 扩展是否启用。

## 10. 核心原理解释

Laravel 是 PHP 框架，Composer 类似 Java 的 Maven/Gradle。React、TypeScript、Vite 属于前端工具链，由 Node.js 和 npm 管理。MySQL 8 是 TaskHub 的业务数据库。

`pdo_mysql` 是 Laravel 连接 MySQL 的关键扩展。如果没有它，Laravel 会启动但无法连接数据库。

`redis` 是 PHP 连接 Redis 的扩展。TaskHub 当前不把 Session 或 Queue 放进 Redis，只用 Redis 缓存外部付款账号和本据点人员大列表，避免每次请求都慢查询外部接口。

## 11. 与 Spring Boot 的概念对照

| Laravel / PHP | Spring Boot / Java |
|---|---|
| PHP 8.5.8 | JDK |
| Composer | Maven / Gradle |
| `composer.json` | `pom.xml` / `build.gradle` |
| PHP extension | JVM 依赖或本地驱动 |
| MySQL CLI | MySQL 客户端 |
| Redis | Redis / Spring Cache 常用后端 |

## 13. 常见错误与解决方法

| 错误 | 原因 | 处理 |
|---|---|---|
| `php: command not found` | PHP 未安装或 PATH 未配置 | 安装 PHP 并配置 PATH |
| `composer: command not found` | Composer 未安装 | 安装 Composer |
| `pdo_mysql` 不存在 | PHP MySQL 扩展未启用 | 启用扩展后重启终端 |
| `redis` 扩展不存在 | PHP Redis 扩展未启用 | 安装或启用 phpredis 扩展后重启终端 |
| `mysql: command not found` | MySQL CLI 未安装 | 安装 MySQL client |
| `redis-server: command not found` | Redis 未安装或 PATH 未配置 | 安装 Redis 或连接远程 Redis |
| PHP 版本不是 8.5.8 | 本机默认 PHP 版本不对 | 切换 PHP 版本 |
