# 03-数据库配置

## 1. 本章节目标

本章说明 Laravel 如何连接已有 MySQL 8 数据库，以及为什么本阶段不执行 migration。

TaskHub 的数据库标准是 `database/schema.sql`。代码必须服从 schema；如果发现冲突，先提出建议，不直接改数据库。

## 2. 涉及文件

| 文件 | 作用 |
|---|---|
| `database/schema.sql` | 当前唯一数据库结构标准 |
| `.env.example` | MySQL 连接环境变量模板 |
| `config/database.php` | Laravel 数据库连接配置 |
| `database/seeders/DatabaseSeeder.php` | Seeder 占位，不写入默认用户 |

## 3. 实现步骤

1. `.env.example` 默认使用 `DB_CONNECTION=mysql`。
2. `config/database.php` 默认连接改为 MySQL。
3. 删除 Laravel 默认 `users/cache/jobs` migration。
4. Seeder 不创建用户或演示数据。
5. Eloquent Model 使用当前表名和字段名映射现有 schema。

## 4. 核心代码讲解

Laravel 的数据库配置读取 `.env`：

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=taskhub
DB_USERNAME=root
DB_PASSWORD=
```

这类似 Spring Boot：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/taskhub
```

Eloquent Model 通过 `$table` 指定表名，通过 `$fillable` 控制可批量赋值字段，通过 `casts()` 控制 JSON、日期、decimal 的 PHP 表示。

## 5. 设计原因

不使用 migration 改数据库的原因：

- 数据库结构已经由 `database/schema.sql` 确认。
- 默认 Laravel migration 会创建 `users`、`cache`、`jobs`，与当前方案冲突。
- Phase 1 目标是映射已有结构，不重新设计结构。

后续如果需要 migration，应先确认它是否只是为了部署初始化；任何结构变更仍要先修改并评审 `schema.sql`。

## 6. 注意事项

- 不要创建本地 `users` 表。
- 不要执行默认 migration。
- 金额字段在 Model 中 cast 为 `decimal:2`。
- JSON 快照字段 cast 为 `array`。
- 工号字段是 `VARCHAR(32)`，不要当成本地用户外键。

## 7. 本阶段完成效果

完成后代码具备连接 MySQL 8 的配置，且没有任何会创建无关表的 migration。可以用以下命令检查配置：

```bash
php artisan config:show database.default
php artisan route:list
```
