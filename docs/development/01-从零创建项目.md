# 01-从零创建项目

## 1. 本章目标

本章从一个空目录开始，创建一个与当前 TaskHub Phase 1 基础架构一致的 Laravel 13 + React + TypeScript + Inertia.js 项目。

读完并操作完成后，你应该理解：

- Laravel 负责路由、配置、服务端渲染入口和后端代码组织。
- Blade 在本项目中只作为 React/Inertia 的 HTML 容器，不承载业务页面。
- Inertia 负责把 Laravel 路由返回的页面名和 props 交给 React。
- React 负责渲染浏览器中的页面组件。
- Vite 负责开发环境热更新、TypeScript/React 编译和生产构建。

本章面向 Java 开发者，所以会把 Laravel 概念和 Spring Boot 中的常见概念做对照。

## 2. 前置条件

先完成 [00-开发环境准备](./00-开发环境准备.md)，至少确认以下命令可用：

```bash
php -v
composer --version
node -v
npm -v
mysql --version
```

TaskHub 当前要求 PHP 最低版本为 `8.5.8`，MySQL 使用 8.x。

## 3. 本章最终效果

完成后，在项目根目录运行：

```bash
composer run dev
```

访问：

```text
http://127.0.0.1:8000
```

浏览器应看到 React 页面，页面中包含 TaskHub 应用名。浏览器 Console 不应出现 TypeScript、Vite 或 Inertia 页面解析错误。

## 4. 创建 Laravel 项目

### 4.1 命令执行目录

在你希望放置项目的父目录执行，不要在已经克隆好的 TaskHub 根目录中执行。

正确示例：

```bash
cd ~/Projects
composer create-project laravel/laravel task-hub "13.*"
cd task-hub
```

错误示例：

```bash
cd ~/Projects/task-hub
composer create-project laravel/laravel task-hub "13.*"
```

上面的错误命令会生成 `task-hub/task-hub` 嵌套目录。

### 4.2 命令作用

`composer create-project laravel/laravel task-hub "13.*"` 会下载 Laravel 13 官方项目骨架，类似 Java 中通过 Spring Initializr 创建一个 Spring Boot 项目。

生成的关键文件和目录包括：

```text
artisan
app/
bootstrap/
config/
database/
public/
resources/
routes/
tests/
composer.json
package.json
vite.config.js
```

### 4.3 立即检查版本

命令执行目录：新建的 `task-hub` 项目根目录。

```bash
php -v
php artisan --version
```

预期结果：

- `php -v` 显示 PHP `8.5.8` 或更高兼容版本。
- `php artisan --version` 显示 Laravel Framework 13.x。

如果 `php artisan --version` 报 `vendor/autoload.php` 不存在，说明依赖没有安装完整，可以执行：

```bash
composer install
```

## 5. 安装后端依赖

命令执行目录：项目根目录。

```bash
composer require inertiajs/inertia-laravel
composer remove laravel/pail --dev
composer require pestphp/pest --dev --with-all-dependencies
```

### 5.1 Inertia Laravel

`composer require inertiajs/inertia-laravel` 会自动修改：

```text
composer.json
composer.lock
```

不需要手工把 `inertiajs/inertia-laravel` 写入 `composer.json`。

它的职责类似 Spring MVC 中“返回视图模型”的桥接层：Laravel 路由返回页面名和数据，Inertia 把它们交给前端 React 页面。

### 5.2 删除 Laravel Pail

Pail 是 Laravel 的实时日志查看开发工具。

TaskHub Phase 1 暂时不需要实时日志查看工具，所以删除：

```bash
composer remove laravel/pail --dev
```

这和 Laravel Auth 无关。TaskHub 不使用本地 users 表和默认认证体系，但删除 Pail 的原因只是当前阶段用不到该开发工具。

### 5.3 安装 Pest

Pest 是 PHP 测试框架，当前项目使用它运行基础测试。

```bash
composer require pestphp/pest --dev --with-all-dependencies
```

它和 Java 中的 JUnit 角色相近，但语法更简洁。

## 6. 安装前端依赖

命令执行目录：项目根目录。

```bash
npm install react react-dom @inertiajs/react
npm install -D typescript @types/react @types/react-dom @vitejs/plugin-react
```

检查 Tailwind 和 Laravel Vite 插件是否已经存在：

```bash
npm ls tailwindcss @tailwindcss/vite laravel-vite-plugin
```

如果缺少 Tailwind 4 相关依赖，再执行：

```bash
npm install -D tailwindcss @tailwindcss/vite
```

当前项目的 `composer run dev` 使用 `concurrently` 同时启动 Laravel 和 Vite。如果你的骨架中没有该依赖，执行：

```bash

```

| 依赖 | 职责 |

## 7. 调整 `composer.json`

### 7.1 PHP 版本约束

文件：`composer.json`

当前 TaskHub 的关键配置：

```json
"require": {
    "php": "^8.5.8",
    "inertiajs/inertia-laravel": "^3.1",
    "laravel/framework": "^13.8",
    "laravel/tinker": "^3.0"
}
```

`"php": "^8.5.8"` 表示最低使用 PHP 8.5.8，并允许兼容的后续 8.x 版本。

### 7.2 `setup` 脚本修改前

Laravel 官方骨架可能包含类似脚本：

```json
"setup": [
    "composer install",
    "@php -r \"file_exists('.env') || copy('.env.example', '.env');\"",
    "@php artisan key:generate",
    "@php artisan migrate --force",
    "npm install",
    "npm run build"
]
```

### 7.3 `setup` 脚本修改后

TaskHub 当前使用：

```json
"setup": [
    "composer install",
    "@php -r \"file_exists('.env') || copy('.env.example', '.env');\"",
    "@php artisan key:generate",
    "npm install --ignore-scripts",
    "npm run build"
]
```

删除 `php artisan migrate --force` 的原因：

- TaskHub 当前以 `database/schema.sql` 作为数据库结构标准。
- Phase 1 不使用 Laravel Migration 修改业务表结构。
- 项目不创建本地 `users` 表。
- 自动执行 migration 可能创建 Laravel 默认无关表，和当前数据库设计不一致。

### 7.4 `dev` 脚本

TaskHub 当前脚本：

```json
"dev": [
    "Composer\\Config::disableProcessTimeout",
    "npx concurrently -c \"#93c5fd,#fb7185\" \"php artisan serve\" \"npm run dev\" --names=server,vite --kill-others"
]
```

作用：

- `php artisan serve` 启动 Laravel 开发服务器。
- `npm run dev` 启动 Vite 开发服务器。
- `concurrently` 把两个进程放在一个命令里运行。

## 8. 逐文件创建和修改

### 8.1 创建目录、TypeScript 类型文件和 CSS 文件

命令执行目录：项目根目录。

```bash
mkdir -p resources/js/Pages
mkdir -p resources/css
mkdir -p app/Http/Middleware
```

Laravel 默认已有 `resources/views`、`routes`、`bootstrap` 等目录。

接下来先补齐 TypeScript、Vite 和 CSS 的基础文件。原因是 `app.tsx` 会用到 Vite 的 `import.meta.glob()`，并且会导入 `../css/app.css`。如果类型文件或 CSS 文件缺失，会出现两个常见错误：

```text
Property 'glob' does not exist on type 'ImportMeta'
Cannot find module '../css/app.css'
```

#### 8.1.1 配置 `tsconfig.json`

文件：`tsconfig.json`

完整内容：

```json
{
    "compilerOptions": {
        "allowJs": false,
        "allowSyntheticDefaultImports": true,
        "jsx": "react-jsx",
        "lib": ["DOM", "DOM.Iterable", "ES2023"],
        "module": "ESNext",
        "moduleResolution": "Bundler",
        "noEmit": true,
        "resolveJsonModule": true,
        "skipLibCheck": true,
        "strict": true,
        "target": "ES2023",
        "types": ["vite/client"]
    },
    "include": ["resources/js/**/*.ts", "resources/js/**/*.tsx", "resources/js/**/*.d.ts"]
}
```

关键说明：

- `"types": ["vite/client"]`：加载 Vite 提供的浏览器端类型声明，其中包含 `import.meta.glob()`。
- `"include"` 必须包含 `resources/js/**/*.d.ts`，否则后面创建的 `vite-env.d.ts` 可能不会被 TypeScript 读取。
- `"moduleResolution": "Bundler"` 更贴近 Vite 的模块解析方式。
- `"jsx": "react-jsx"` 对应 React 17 以后推荐的 JSX 转换方式，不需要每个 TSX 文件都 `import React from 'react'`。

如果 VS Code 仍然提示 `Property 'glob' does not exist on type 'ImportMeta'`，但命令行 `npm run typecheck` 已经通过，通常是编辑器 TypeScript 服务缓存了旧配置。处理方式：

```text
Command Palette
→ TypeScript: Restart TS Server
```

如果仍未恢复，关闭并重新打开 VS Code 工作区。

#### 8.1.2 创建 Vite 类型声明

文件：`resources/js/vite-env.d.ts`

完整内容：

```ts
/// <reference types="vite/client" />
```

这个文件是 Vite 官方常见写法，用来确保当前前端源码目录能识别 Vite 注入的类型。它和 `tsconfig.json` 中的 `"types": ["vite/client"]` 是同一件事的双保险：前者靠文件引用，后者靠编译配置。

验证方式：

```bash
npm run typecheck
```

此时如果还没有创建 `app.tsx`，可能不会检查到 `import.meta.glob()`；等 8.3 完成后需要再运行一次。

#### 8.1.3 创建全局 CSS 文件

文件：`resources/css/app.css`

完整内容：

```css
@import 'tailwindcss';

@source '../../vendor/laravel/framework/src/Illuminate/Pagination/resources/views/*.blade.php';
@source '../../storage/framework/views/*.php';

@theme {
    --font-sans: 'Instrument Sans', ui-sans-serif, system-ui, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji',
        'Segoe UI Symbol', 'Noto Color Emoji';
}
```

这个文件由 `resources/js/app.tsx` 导入：

```tsx
import '../css/app.css';
```

路径解释：

- `app.tsx` 位于 `resources/js/app.tsx`。
- `../css/app.css` 表示从 `resources/js` 回到 `resources`，再进入 `css/app.css`。

因此 CSS 文件必须创建在 `resources/css/app.css`，不是 `resources/js/css/app.css`。

验证方式：

```bash
test -f resources/css/app.css && echo "app.css exists"
```

Windows PowerShell：

```powershell
Test-Path resources/css/app.css
```

### 8.2 创建 Inertia 根 Blade

文件：`resources/views/app.blade.php`

完整内容：

```blade
<!DOCTYPE html>
<html lang="{{ str_replace('_', '-', app()->getLocale()) }}">
    <head>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1">
        <title inertia>{{ config('app.name', 'TaskHub') }}</title>
        @viteReactRefresh
        @vite('resources/js/app.tsx')
        @inertiaHead
    </head>
    <body>
        @inertia
    </body>
</html>
```

Blade 是 Laravel 的服务端模板引擎，类似 Spring MVC 中的 Thymeleaf 或 JSP。

TaskHub 虽然使用 React，但仍需要这个 Blade 文件，原因是浏览器第一次访问 Laravel 路由时，Laravel 必须返回一个 HTML 文档。这个 Blade 不是业务页面，只是 React/Inertia 的 HTML 容器。

每条指令的作用：

- `<!DOCTYPE html>`：告诉浏览器使用标准 HTML 模式。
- `<html lang="...">`：设置页面语言。
- `<meta charset="utf-8">`：支持中文等字符。
- `<meta name="viewport">`：保证移动端宽度正常。
- `<title inertia>`：允许 Inertia 后续更新页面标题。
- `@viteReactRefresh`：开发环境启用 React Fast Refresh。
- `@vite('resources/js/app.tsx')`：加载 Vite 编译的 React 入口。
- `@inertiaHead`：让 React 页面可以注入 title、meta 等 head 内容。
- `@inertia`：输出 Inertia 根节点，React 会挂载到这里。

统一 CSS 加载方案：

- Blade 只加载 `resources/js/app.tsx`。
- CSS 由 `app.tsx` 中的 `import '../css/app.css';` 加载。
- 因此不要在 Blade 里同时写 CSS 和 JS 两个 Vite 入口。
- Vite 也不要把 CSS 配成单独入口，否则同一份 CSS 会出现多处入口，初学者很难判断到底由谁加载。

验证方式：

```bash
php artisan view:clear
php artisan route:list
```

此时路由可能还没有改成 Inertia 页面，`route:list` 只是确认 Laravel 仍能启动并解析应用。

### 8.3 创建 React 入口

文件：`resources/js/app.tsx`

完整内容：

```tsx
import '../css/app.css';

import { createInertiaApp } from '@inertiajs/react';
import type { ComponentType } from 'react';
import { createRoot } from 'react-dom/client';

type PageModule = {
    default: ComponentType;
};

const pages = import.meta.glob<PageModule>('./Pages/**/*.tsx');

createInertiaApp({
    title: (title) => (title ? `${title} - TaskHub` : 'TaskHub'),
    resolve: async (name) => {
        const page = pages[`./Pages/${name}.tsx`];

        if (!page) {
            throw new Error(`Page not found: ${name}`);
        }

        return (await page()).default;
    },
    setup({ el, App, props }) {
        if (!el) {
            throw new Error('Inertia root element was not found.');
        }

        createRoot(el).render(<App {...props} />);
    },
});
```

这相当于 Java Web 项目中的前端启动入口。

关键说明：

- `import '../css/app.css';`：把全局 CSS 纳入 Vite 构建。
- `createInertiaApp()`：创建 Inertia React 应用。
- `ComponentType`：React 组件类型声明，帮助 TypeScript 理解页面模块。
- `PageModule`：约束每个页面模块必须有 `default` 导出的 React 组件。
- `import.meta.glob('./Pages/**/*.tsx')`：Vite 提供的文件扫描能力，把 `Pages` 目录下的页面组件建立成映射表。
- `pages` 放在 `createInertiaApp` 外部：页面映射只声明一次，不需要每次 `resolve` 都重新创建。
- `resolve(name)`：把 Laravel 返回的页面名转换成实际 TSX 文件。
- `setup({ el, App, props })`：找到 Blade 中的 Inertia 根节点，把 React 应用挂载进去。
- `props`：来自 Laravel `Inertia::render()` 的第二个参数。

页面名映射示例：

```text
Inertia::render('Welcome')
→ resources/js/Pages/Welcome.tsx

Inertia::render('Tasks/Index')
→ resources/js/Pages/Tasks/Index.tsx
```

注意：目录名使用 `Pages`，不是 `pages`。Linux 服务器大小写敏感，大小写写错会导致本地 macOS 可用、线上不可用。

验证方式：

```bash
npm run typecheck
```

如果这里出现：

```text
Property 'glob' does not exist on type 'ImportMeta'
```

优先检查：

- `resources/js/vite-env.d.ts` 是否存在。
- 文件内容是否是 `/// <reference types="vite/client" />`。
- `tsconfig.json` 是否包含 `"types": ["vite/client"]`。
- `tsconfig.json` 的 `"include"` 是否包含 `resources/js/**/*.d.ts`。
- VS Code 是否需要执行 `TypeScript: Restart TS Server`。

如果出现：

```text
Cannot find module '../css/app.css'
```

优先检查：

- `resources/css/app.css` 是否存在。
- `app.tsx` 是否位于 `resources/js/app.tsx`。
- import 路径是否正好是 `../css/app.css`。

### 8.4 创建最小 React 页面

学习用最小文件：`resources/js/Pages/Welcome.tsx`

```tsx
type WelcomeProps = {
    appName: string;
};

export default function Welcome({ appName }: WelcomeProps) {
    return (
        <main>
            <h1>{appName}</h1>
            <p>Laravel、Inertia 和 React 已连接成功。</p>
        </main>
    );
}
```

当前 TaskHub 实际文件在这个基础上增加了 Tailwind 样式，用于让首页更像一个基础项目验证页，但没有加入任何任务业务功能。

当前 TaskHub 实际文件完整内容：

```tsx
type WelcomeProps = {
    appName: string;
};

export default function Welcome({ appName }: WelcomeProps) {
    return (
        <main className="min-h-screen bg-zinc-950 text-zinc-100">
            <section className="mx-auto flex min-h-screen max-w-5xl flex-col justify-center px-6 py-16">
                <p className="text-sm font-medium uppercase tracking-[0.24em] text-cyan-300">
                    Phase 1
                </p>
                <h1 className="mt-4 text-4xl font-semibold tracking-normal sm:text-5xl">
                    {appName}
                </h1>
                <p className="mt-5 max-w-2xl text-base leading-7 text-zinc-300">
                    Laravel 13 monolith with React, TypeScript, Inertia.js, MySQL 8, and Pest.
                    This screen only verifies the application shell.
                </p>
            </section>
        </main>
    );
}
```

数据传递关系：

Laravel 路由中写：

```php
'appName' => config('app.name')
```

React Props 中写：

```tsx
type WelcomeProps = {
    appName: string;
};
```

两边字段名必须一致。`appName` 在 PHP 数组中是 key，经过 Inertia 序列化后，会成为 React 组件的参数。

完整链路：

```text
PHP 数组
→ Inertia 序列化为页面 props
→ TypeScript WelcomeProps 描述 props 结构
→ React 组件参数 { appName }
```

验证方式：

```bash
npm run typecheck
```

如果 `appName` 在 PHP 中写成 `app_name`，但 React 中仍然写 `appName`，页面运行时会拿不到正确值。Inertia 不会自动把 snake_case 改成 camelCase。

### 8.5 配置 Vite

文件：`vite.config.ts`

如果骨架生成的是 `vite.config.js`，把它改名为 `vite.config.ts`。

当前 TaskHub 完整内容：

```ts
import react from '@vitejs/plugin-react';
import tailwindcss from '@tailwindcss/vite';
import { defineConfig } from 'vite';
import { bunny } from 'laravel-vite-plugin/fonts';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
    plugins: [
        laravel({
            input: ['resources/js/app.tsx'],
            refresh: true,
            fonts: [
                bunny('Instrument Sans', {
                    weights: [400, 500, 600],
                }),
            ],
        }),
        react(),
        tailwindcss(),
    ],
    server: {
        watch: {
            ignored: ['**/storage/framework/views/**'],
        },
    },
});
```

核心配置说明：

- `laravel-vite-plugin`：让 Laravel 的 `@vite()` 能找到 Vite 开发服务和生产构建产物。
- `input: ['resources/js/app.tsx']`：唯一入口是 React 入口文件。
- CSS 不单独作为 Vite 入口，因为已经由 `app.tsx` import。
- `refresh: true`：修改 PHP/Blade 等文件时自动刷新浏览器。
- `@vitejs/plugin-react`：让 Vite 支持 React 和 TSX。
- `@tailwindcss/vite`：让 Vite 支持 Tailwind 4。
- `server.watch.ignored`：忽略 Laravel 编译后的视图缓存，减少不必要刷新。

当前项目额外保留 Bunny Fonts 配置：

```ts
fonts: [
    bunny('Instrument Sans', {
        weights: [400, 500, 600],
    }),
],
```

它只是项目样式配置，不是 Inertia 运行的必要条件。为了降低学习难度，你也可以先不加字体配置，等页面跑通后再补。

验证方式：

```bash
npm run build
```

构建输出中应该能看到 `public/build/manifest.json`，并且 manifest 中有 `resources/js/app.tsx` 入口。

### 8.6 配置 Inertia Middleware

文件：`app/Http/Middleware/HandleInertiaRequests.php`

完整内容：

```php
<?php

namespace App\Http\Middleware;

use Illuminate\Http\Request;
use Inertia\Middleware;

class HandleInertiaRequests extends Middleware
{
    protected $rootView = 'app';

    public function share(Request $request): array
    {
        return [
            ...parent::share($request),
            'app' => [
                'name' => config('app.name'),
            ],
        ];
    }
}
```

说明：

- `$rootView = 'app'` 对应 `resources/views/app.blade.php`。
- `share()` 用于给所有 Inertia 页面共享公共数据。
- 当前共享应用名，未来可以共享当前登录用户、菜单信息等公共数据。
- 它类似 Spring MVC 中的全局模型属性或拦截器中注入公共上下文。

注册 Middleware。

文件：`bootstrap/app.php`

顶部增加：

```php
use App\Http\Middleware\HandleInertiaRequests;
```

在 `->withMiddleware()` 中追加：

```php
->withMiddleware(function (Middleware $middleware): void {
    $middleware->web(append: [
        HandleInertiaRequests::class,
    ]);
})
```

为什么注册到 `web` middleware：

- Inertia 页面走浏览器访问，需要 Session、Cookie、CSRF、页面响应等 Web 能力。
- TaskHub 是 Laravel Monolith + Inertia，不是纯 API 项目。
- 注册到 `web` 后，所有 Web 页面都能获得 Inertia 的共享数据和根视图配置。

验证方式：

```bash
php artisan route:list
```

如果出现 `Class "App\Http\Middleware\HandleInertiaRequests" not found`，检查：

- 文件路径是否是 `app/Http/Middleware/HandleInertiaRequests.php`。
- namespace 是否是 `App\Http\Middleware`。
- `bootstrap/app.php` 顶部是否有 `use App\Http\Middleware\HandleInertiaRequests;`。

### 8.7 配置第一个 Inertia Route

文件：`routes/web.php`

完整内容：

```php
<?php

use Inertia\Inertia;
use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return Inertia::render('Welcome', [
        'appName' => config('app.name'),
    ]);
});
```

说明：

- `Route::get('/')`：定义浏览器访问首页时的处理逻辑，类似 Spring Boot 中 `@GetMapping("/")`。
- `Inertia::render('Welcome')`：返回页面名，不再返回传统 Blade 业务页面。
- 第二个参数是 props，会传给 React 页面。
- `config('app.name')` 从 Laravel 配置读取应用名。

为什么不再返回 `welcome.blade.php`：

- 默认 `welcome.blade.php` 是 Laravel 官方欢迎页。
- TaskHub 的业务页面由 React 组件负责。
- `app.blade.php` 是容器，`Welcome.tsx` 才是页面。

`Inertia::render('Welcome')` 最终会打开 `resources/js/Pages/Welcome.tsx`，因为 `app.tsx` 中的 `resolve()` 使用了下面的映射规则：

```tsx
const page = pages[`./Pages/${name}.tsx`];
```

所以：

```text
Welcome
→ ./Pages/Welcome.tsx
→ resources/js/Pages/Welcome.tsx
```

验证方式：

```bash
php artisan route:list
npm run typecheck
```

## 9. 所有文件如何连接

这是理解 Laravel + Inertia + React 的关键。

完整流程：

```text
┌────────────┐
│ 浏览器访问 / │
└─────┬──────┘
      ↓
┌────────────────┐
│ routes/web.php │
└─────┬──────────┘
      ↓
┌──────────────────────────────┐
│ Inertia::render('Welcome')   │
│ props: ['appName' => ...]    │
└─────┬────────────────────────┘
      ↓
┌──────────────────────────────┐
│ HandleInertiaRequests        │
│ rootView = app               │
│ share common props           │
└─────┬────────────────────────┘
      ↓
┌──────────────────────────────┐
│ resources/views/app.blade.php│
│ @vite + @inertia             │
└─────┬────────────────────────┘
      ↓
┌──────────────────────────────┐
│ resources/js/app.tsx         │
│ createInertiaApp + resolve   │
└─────┬────────────────────────┘
      ↓
┌───────────────────────────────┐
│ resources/js/Pages/Welcome.tsx│
│ React props → UI              │
└─────┬─────────────────────────┘
      ↓
┌────────────┐
│ React 页面 │
└────────────┘
```

逐步说明：

1. 浏览器访问 `http://127.0.0.1:8000/`。
2. Laravel 在 `routes/web.php` 中匹配 `Route::get('/')`。
3. 路由返回 `Inertia::render('Welcome', ['appName' => config('app.name')])`。
4. `HandleInertiaRequests` 告诉 Inertia 根 Blade 是 `app`，也就是 `resources/views/app.blade.php`。
5. Laravel 返回 `app.blade.php` 生成的 HTML 容器。
6. Blade 中的 `@vite('resources/js/app.tsx')` 加载 Vite 编译后的 React 入口。
7. `app.tsx` 调用 `createInertiaApp()`，根据页面名 `Welcome` 找到 `resources/js/Pages/Welcome.tsx`。
8. Inertia 把 PHP props 序列化后交给 React。
9. React 执行 `Welcome({ appName })` 并渲染页面。

各文件职责：

| 文件 | 职责 | Spring Boot 类比 |
|---|---|---|
| `routes/web.php` | 定义 Web 路由 | Controller 路由方法 |
| `Inertia::render()` | 返回页面名和 props | 返回 ModelAndView 或 DTO |
| `HandleInertiaRequests.php` | Inertia 全局配置和共享 props | HandlerInterceptor / ControllerAdvice |
| `app.blade.php` | HTML 容器 | Thymeleaf 根模板 |
| `app.tsx` | React/Inertia 启动入口 | 前端 main 方法 |
| `Pages/Welcome.tsx` | React 页面组件 | 前端页面类 |
| `vite.config.ts` | 前端构建配置 | Maven/Gradle 前端构建插件配置 |

## 10. 核心原理解释

Laravel 传统模式是：

```text
Route
→ Controller
→ Blade 页面
→ HTML
```

TaskHub 当前模式是：

```text
Route
→ Inertia 页面名 + props
→ React 页面
→ HTML
```

它不是完全前后端分离。浏览器仍访问 Laravel 路由，认证、Session、CSRF、服务端配置都仍在 Laravel 内完成。

它也不是传统 Blade 项目。业务页面不写在 Blade 中，而写在 React TSX 文件中。

这种方式适合 TaskHub Phase 1：

- 保持 Laravel 单体架构，部署简单。
- 前端使用 React + TypeScript，适合后续复杂交互。
- 不需要维护独立 API 项目和独立前端项目。
- 不引入 DDD、Repository、微服务、队列等当前不需要的复杂设计。

## 11. 与 Spring Boot 的概念对照

| Laravel 概念 | Spring Boot 类比 | 差异 |
|---|---|---|
| `artisan` | `mvn spring-boot:run` / Gradle task | Laravel 命令入口 |
| `routes/web.php` | `@Controller` + `@GetMapping` | Laravel 可以直接在路由文件写闭包 |
| Middleware | Filter / Interceptor | Laravel middleware 按 web/api 分组 |
| Blade | Thymeleaf / JSP | 本项目只把 Blade 当 HTML 容器 |
| Inertia props | Model attributes / DTO | 会直接传给 React |
| Eloquent Model | JPA Entity | 后续使用，但本章不开发业务 |
| Vite | 前端构建插件 | 负责 React/TS 编译和热更新 |
| Pest | JUnit | PHP 测试框架 |

## 13. 常见错误与解决方法

| 错误 | 常见原因 | 处理 |
|---|---|---|
| `Vite manifest not found` | 没有启动 Vite，也没有执行生产构建 | 开发时运行 `composer run dev`；生产检查先运行 `npm run build` |
| `Unable to locate file in Vite manifest` | `@vite()` 入口和 `vite.config.ts` input 不一致 | 确认两处都是 `resources/js/app.tsx` |
| `Page not found: Welcome` | 页面文件路径和 `Inertia::render()` 名称不一致 | 确认文件是 `resources/js/Pages/Welcome.tsx` |
| `Middleware class not found` | `bootstrap/app.php` import 或命名空间错误 | 确认 `use App\Http\Middleware\HandleInertiaRequests;` 存在 |
| React 页面空白 | React 没挂载或浏览器 JS 报错 | 看 Console，检查 `@inertia`、`app.tsx`、`createRoot` |
| Blade 指令被当成普通文本 | 文件不是 `.blade.php` 或没有经过 Laravel 渲染 | 确认文件名是 `resources/views/app.blade.php`，通过 Laravel 路由访问 |
| `Pages/pages` 大小写错误 | Linux 大小写敏感 | 统一使用 `resources/js/Pages` |
| Tailwind 不生效 | CSS 没被入口加载或插件缺失 | 确认 `app.tsx` import `../css/app.css`，且 Vite 有 `tailwindcss()` |
| TypeScript 编译错误 | 类型依赖缺失或页面模块类型错误 | 执行 `npm install`，检查 `PageModule` 和页面默认导出 |
| `Property 'glob' does not exist on type 'ImportMeta'` | Vite 类型没有加载，或 VS Code TypeScript 服务缓存旧配置 | 检查 `tsconfig.json`、`resources/js/vite-env.d.ts`，然后执行 `TypeScript: Restart TS Server` |
| `Cannot find module '../css/app.css'` | CSS 文件缺失，或 `vite/client` 类型未生效 | 确认 `resources/css/app.css` 存在，确认 `vite-env.d.ts` 和 `tsconfig.json` 正确 |
| 首页 500 且提示数据库连接 | `.env` 数据库配置不正确 | 本章首页不需要业务表，但 Laravel 配置仍可能连接数据库，按第 04 章配置 |
