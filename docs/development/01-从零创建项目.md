# 01-从零创建项目

## 1. 本章目标

从空目录复现 TaskHub Phase 1 的 Laravel 13 + React + TypeScript + Inertia.js 项目骨架。

## 2. 前置条件

已完成 [00-开发环境准备](./00-开发环境准备.md)。

## 3. 本章最终效果

你可以在浏览器打开 Laravel 页面，并看到 React 通过 Inertia 渲染的最小页面。

## 4. 实际执行命令

```bash
composer create-project laravel/laravel task-hub "13.*"
cd task-hub
composer require inertiajs/inertia-laravel
composer remove laravel/pail --dev
composer require pestphp/pest --dev --with-all-dependencies
npm install react react-dom @inertiajs/react
npm install -D typescript @types/react @types/react-dom @vitejs/plugin-react
```

## 5. 命令执行目录

- `composer create-project` 在你希望创建项目的父目录执行。
- 其余命令在新建的 `task-hub` 项目根目录执行。

## 6. 命令作用

- `create-project`：生成 Laravel 13 官方骨架。
- `inertiajs/inertia-laravel`：让 Laravel 能返回 Inertia 页面。
- `composer remove laravel/pail`：当前不使用 Laravel Auth，本项目移除依赖 Auth guard 的日志工具。
- `pestphp/pest`：安装 Pest 测试框架。
- React 相关 npm 包：提供 Inertia 前端渲染能力。

## 8. 逐文件修改过程

### 8.1 修改 `composer.json`

修改前关键片段：

```json
"require": {
  "php": "^8.3",
  "laravel/framework": "^13.8"
}
```

修改后：

```json
"require": {
  "php": "^8.5.8",
  "inertiajs/inertia-laravel": "^3.1",
  "laravel/framework": "^13.8"
}
```

同时删除 `setup` 中的 `php artisan migrate --force`，避免初始化时误改已有数据库。

### 8.2 创建 Inertia 根 Blade

创建 `resources/views/app.blade.php`：

```blade
@viteReactRefresh
@vite(['resources/css/app.css', 'resources/js/app.tsx'])
@inertiaHead
@inertia
```

这不是传统业务 Blade 页面，只是 React/Inertia 的挂载壳。

### 8.3 创建 React 入口

创建 `resources/js/app.tsx`：

```tsx
createInertiaApp({
    resolve: async (name) => {
        const pages = import.meta.glob<PageModule>('./Pages/**/*.tsx');
        return (await pages[`./Pages/${name}.tsx`]()).default;
    },
    setup({ el, App, props }) {
        createRoot(el).render(<App {...props} />);
    },
});
```

### 8.4 创建最小页面

创建 `resources/js/Pages/Welcome.tsx`，接收 `appName` props 并展示项目名。

### 8.5 配置 Vite

将 `vite.config.js` 改为 `vite.config.ts`，并加入 React 插件：

```ts
plugins: [
    laravel({ input: ['resources/css/app.css', 'resources/js/app.tsx'] }),
    react(),
    tailwindcss(),
]
```

### 8.6 配置 Inertia Middleware

创建 `app/Http/Middleware/HandleInertiaRequests.php`，设置：

```php
protected $rootView = 'app';
```

并在 `bootstrap/app.php` 注册到 web middleware。

### 8.7 配置第一个 Inertia Route

修改 `routes/web.php`：

```php
Route::get('/', fn () => Inertia::render('Welcome', [
    'appName' => config('app.name'),
]));
```

### 8.8 删除默认文件

删除或不提交：

- `app/Models/User.php`
- `database/factories/UserFactory.php`
- 默认 `database/migrations/*users/cache/jobs*`
- `resources/views/welcome.blade.php`

原因：TaskHub 不创建本地 users 表，不使用默认 Blade 欢迎页，不通过 migration 创建无关表。

## 9. 修改前后的关键代码

`routes/web.php` 修改前：

```php
Route::get('/', function () {
    return view('welcome');
});
```

修改后：

```php
Route::get('/', function () {
    return Inertia::render('Welcome', [
        'appName' => config('app.name'),
    ]);
});
```

## 10. 核心原理解释

Inertia 让 Laravel Route 直接返回 React 页面名和 props。浏览器仍访问 Laravel 路由，但页面组件由 React 渲染。

## 11. 与 Spring Boot 的概念对照

| Laravel + Inertia | Spring Boot 类比 |
|---|---|
| `routes/web.php` | Controller 路由 |
| `Inertia::render()` | 返回视图模型 |
| React Page | 前端页面组件 |
| `app.blade.php` | SPA HTML 容器 |
| Vite | 前端构建插件 |

## 13. 常见错误与解决方法

| 错误 | 处理 |
|---|---|
| `Page not found` | 检查 `resources/js/Pages/{Name}.tsx` 路径和 `Inertia::render()` 名称 |
| Vite 入口找不到 | 检查 `vite.config.ts` input 是否是 `resources/js/app.tsx` |
| TypeScript 报 import 类型错误 | 给 `import.meta.glob` 指定模块类型 |
| 首页 500 且提示 SQLite | 检查 `.env` 是否仍是 Laravel 默认 SQLite |
