# 13-开发任务清单

## 本章目标

本章用于记录 TaskHub 开发过程中已经完成和还没有完成的任务。

它不是需求文档的替代品，而是一个项目推进清单。后续每完成一个业务模块，都应该同步更新这里，避免只靠聊天记录或个人记忆判断项目进度。

## 状态说明

| 状态 | 含义 |
|---|---|
| 已完成 | 当前仓库已有代码、配置、测试或文档支撑，可以作为后续开发基础 |
| 部分完成 | 当前已有结构或文档，但业务流程还没有完整实现 |
| 未完成 | 当前还没有正式实现 |
| 暂不做 | 已明确不属于当前 MVP 或 Phase 1 范围 |

## 下一步建议

下一步建议开发“投标功能”。

原因：

- 发布任务已经完成，任务进入 `OPEN` 后自然需要有人投标。
- 数据库已经有 `bid` 和 `bid_member`，可以直接进入业务闭环开发。
- 投标功能会把 TaskHub 从“只能发布和查看任务”推进到“任务开始流转”。

投标功能建议包含：

1. 在任务大厅卡片上增加“投标”按钮。
2. 打开投标模态框。
3. 填写报价、交付日期、方案说明、协作者工号、附件 ID。
4. 后端创建 `bid`。
5. 后端创建 OWNER 和 COLLABORATOR `bid_member`。
6. 后端写入 `BID_SUBMITTED` TaskEvent。
7. 增加 Feature Test 覆盖同一任务同一 OWNER 只能有一条 ACTIVE 投标。
