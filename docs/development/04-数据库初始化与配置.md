# 04-数据库初始化与配置

## 1. 本章目标

使用 `database/schema.sql` 初始化 MySQL 8，并让 Laravel 连接该数据库。

## 2. 前置条件

- MySQL 8 可用。
- 已创建或准备创建 `taskhub` 数据库。
- 已复制 `.env`。

## 3. 本章最终效果

`taskhub` 数据库中存在 8 张核心业务表和 1 张角色控制表，Laravel 可以连接 MySQL。

注意：本章只验证“数据库已经初始化、Laravel 能连接数据库”。此时如果你是从空目录跟着文档操作，还没有 `app/Models/Task.php` 是正常的。Eloquent Model 会在第 07 章创建和讲解。

## 4. 实际执行命令

MySQL CLI：

```bash
mysql -uroot -p < database/schema.sql
mysql -uroot -p -e "SHOW TABLES FROM taskhub;"
php artisan config:clear
php artisan db:show
php artisan tinker
```

Tinker 中执行：

```php
DB::select('select 1 as ok');
```

## 5. 命令执行目录

`mysql -uroot -p < database/schema.sql` 在项目根目录执行，因为 SQL 文件路径是相对路径。

## 6. 命令作用

- `database/schema.sql`：创建 `taskhub` 数据库、8 张核心业务表和 1 张角色控制表。
- `mysql -uroot -p -e "SHOW TABLES FROM taskhub;"`：确认 MySQL 中已经有 TaskHub 表。
- `php artisan config:clear`：清理 Laravel 配置缓存。
- `php artisan db:show`：检查 Laravel 当前数据库连接。
- `php artisan tinker`：进入 Laravel 交互式 REPL，用 Laravel 的数据库连接执行简单 SQL。

## 7. 预期输出或生成文件

`php artisan db:show` 应显示 MySQL 连接和数据库名。

Tinker 中：

```text
> DB::select('select 1 as ok');
= [
    {#... +"ok": 1},
  ]
```

这一步只证明 Laravel 能通过当前 `.env` 连接数据库，不依赖任何 Model。

## 8. 逐文件修改过程

### 8.1 修改 `.env`

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=taskhub
DB_USERNAME=root
DB_PASSWORD=your_password
```

### 8.2 不创建 migration

当前不新增业务 migration。Laravel 默认 users/cache/jobs migration 已删除。

### 8.3 使用 VS Code 数据库插件

如果不用 MySQL CLI，可以：

1. 安装 VS Code 数据库插件。
2. 新建 MySQL 连接。
3. 打开 `database/schema.sql`。
4. 连接到 MySQL 8 后执行整份 SQL。

## 9. 修改前后的关键代码

`.env` 修改前可能是：

```env
DB_CONNECTION=sqlite
```

修改后必须是：

```env
DB_CONNECTION=mysql
DB_DATABASE=taskhub
```

## 10. 核心原理解释

TaskHub 当前以 `database/schema.sql` 作为数据库结构标准。Laravel Migration 暂不用于修改业务结构，避免代码和已确认 SQL 发生分叉。

第 04 章只解决数据库初始化和连接问题。Eloquent Model 只负责映射已存在的表，会在第 07 章开始使用。

## 11. 与 Spring Boot 的概念对照

| Laravel | Spring Boot |
|---|---|
| `.env` DB 配置 | `spring.datasource.*` |
| `php artisan db:show` | 检查 DataSource |
| `php artisan tinker` | JShell / 临时调试 Controller 外代码 |
| `DB::select()` | 使用 JDBC 执行简单 SQL |

## 12. 验证步骤

```bash
php artisan config:clear
php artisan db:show
php artisan tinker
```

Tinker：

```php
DB::select('select 1 as ok');
DB::select('show tables');
```

如果你已经完成第 07 章并创建了 `app/Models/Task.php`，可以再执行：

```php
App\Models\Task::query()->count();
```

但这不是第 04 章的前置验证。

## 13. 常见错误与解决方法

| 错误 | 处理 |
|---|---|
| `Access denied for user` | 检查 `.env` 用户名和密码 |
| `Unknown database taskhub` | 先执行 `database/schema.sql` |
| `could not find driver` | 启用 `pdo_mysql` |
| `.env` 改了不生效 | 执行 `php artisan config:clear` |
| 表不存在 | 确认执行的是当前仓库的 `database/schema.sql` |
| `Class "App\Models\Task" not found` | 这是第 07 章之前的正常情况；第 04 章不要用 Model 验证 |
