# 02-拉取并运行现有项目

## 1. 本章目标

让团队成员从 Git 仓库拉取 TaskHub，并在本地完成依赖安装、环境配置、数据库初始化和首页访问。

## 2. 前置条件

- 已完成 [00-开发环境准备](./00-开发环境准备.md)。
- 有仓库访问权限。
- MySQL 8 可用。

## 3. 本章最终效果

浏览器访问 `http://127.0.0.1:8000` 返回 TaskHub Phase 1 首页。

## 4. 实际执行命令

```bash
git clone <repository-url> task-hub
cd task-hub
composer install
npm install
cp .env.example .env
php artisan key:generate
php artisan config:clear
php artisan cache:clear
php artisan view:clear
```

Windows PowerShell 复制 `.env`：

```powershell
Copy-Item .env.example .env
```

启动：

```bash
composer run dev
```

## 5. 命令执行目录

除 `git clone` 外，所有命令都在 `task-hub` 项目根目录执行。

## 6. 命令作用

- `composer install`：安装 PHP/Laravel 依赖。
- `npm install`：安装 React/Vite/TypeScript 依赖。
- `cp .env.example .env`：创建本地配置文件。
- `php artisan key:generate`：生成 Laravel 加密密钥。
- `php artisan config:clear`：清理配置缓存，确保 `.env` 生效。
- `php artisan cache:clear`：清理应用缓存，避免旧配置或旧缓存影响本地启动。
- `php artisan view:clear`：清理 Blade 编译缓存，避免视图缓存导致页面异常。
- `composer run dev`：同时启动 Laravel 和 Vite。

## 7. 预期输出或生成文件

生成但不提交：

```text
.env
vendor/
node_modules/
public/build/
```

启动成功后应看到：


## 8. 逐文件修改过程

### 8.1 修改 `.env`

从 `.env.example` 复制后，按本机 MySQL 修改：

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=taskhub
DB_USERNAME=root
DB_PASSWORD=
```

### 8.2 检查 `.env` 默认运行配置

继续打开 `.env`，确认至少包含：

```env
SESSION_DRIVER=file
CACHE_STORE=file
QUEUE_CONNECTION=sync
```

这三个配置很重要。它们决定 Laravel 的 Session、Cache 和 Queue 使用什么后端。

#### Session

当前 Phase 1 使用：

```env
SESSION_DRIVER=file
```

原因：

- 不依赖 `sessions` 数据表。
- 不需要执行 Laravel 默认 Session Migration。
- 足够满足当前本地开发和 Inertia 页面访问。

如果误配置为：

```env
SESSION_DRIVER=database
```

Laravel 会去查询：

```text
sessions
```

表。如果数据库中没有这张表，访问首页也会报错：

```text
Illuminate\Database\QueryException
SQLSTATE[42S02]: Base table or view not found:
1146 Table 'taskhub.sessions' doesn't exist
```

当前不要为了修复这个错误去创建 Session Migration，也不要执行 Laravel 默认 migration。TaskHub 当前数据库结构完全以 `database/schema.sql` 为准，不创建 `sessions`、`cache`、`jobs` 这类默认框架表。

#### Cache

当前 Phase 1 使用：

```env
CACHE_STORE=file
```

原因：

- 开发阶段不需要 Redis。
- 不需要 `cache` 数据表。
- 文件缓存足够支持当前基础页面和开发调试。

#### Queue

当前 Phase 1 使用：

```env
QUEUE_CONNECTION=sync
```

原因：

- 当前不启动 Queue Worker。
- 当前没有异步任务。
- `sync` 表示任务在当前请求内同步执行，最容易理解和排查。

### 8.3 清理 Laravel 缓存

修改 `.env` 后必须执行：

```bash
php artisan config:clear
php artisan cache:clear
php artisan view:clear
```

原因：

- Laravel 可能缓存配置，`.env` 改了但运行时还在用旧值。
- `config:clear` 确保新的 `SESSION_DRIVER=file` 生效。
- `cache:clear` 清理应用缓存。
- `view:clear` 清理 Blade 编译缓存。

### 8.4 重新启动开发服务

如果之前已经启动过 `composer run dev`，先按 `Ctrl+C` 停掉，再重新启动：

```bash
composer run dev
```

然后访问：

```text
http://127.0.0.1:8000
```

首页能打开，说明 Laravel、Session、Blade、Vite 和 Inertia 基础链路正常。

SSO 当前保留空值：

```env
SSO_BASE_URL=
SSO_LOGIN_URL=
SSO_VALIDATE_PATH=
```

这些值需要总部协议确认后再填写。

## 9. 修改前后的关键代码

`.env.example` 是模板，`.env` 是本机真实配置。禁止提交 `.env`。

修改前：

```env
DB_PASSWORD=
```

修改后示例：

```env
DB_PASSWORD=your_local_password
```

运行配置必须保持：

```env
SESSION_DRIVER=file
CACHE_STORE=file
QUEUE_CONNECTION=sync
```

## 10. 核心原理解释

Laravel 运行时从 `.env` 读取配置，再由 `config/*.php` 统一暴露给代码。`.env` 类似 Spring Boot 的本地 `application-local.yml`，但 Laravel 默认不会提交它。

TaskHub 当前不使用 Laravel 默认框架表。也就是说：

- Session 不用数据库表，使用文件。
- Cache 不用数据库表，使用文件。
- Queue 不使用数据库队列表，使用同步执行。

这样做是为了让 Phase 1 的数据库结构保持简单，并且严格以 `database/schema.sql` 为准。

## 11. 与 Spring Boot 的概念对照

| Laravel | Spring Boot |
|---|---|
| `.env` | `application-local.yml` |
| `php artisan key:generate` | 初始化应用密钥 |
| `composer install` | `mvn dependency:resolve` |
| `npm install` | 前端依赖安装 |
| `composer run dev` | 同时启动后端和前端 dev server |

## 12. 验证步骤

先执行一次性检查：

```bash
php artisan config:clear
php artisan cache:clear
php artisan view:clear
php artisan route:list
npm run typecheck
npm run build
php artisan test
```

再启动开发服务：

```bash
composer run dev
```

`composer run dev` 是持续运行命令，不会自动退出。保持这个终端打开，另开一个终端验证首页：

```bash
curl -I http://127.0.0.1:8000
```

预期：

```text
HTTP/1.1 200 OK
```

## 13. 常见错误与解决方法

| 错误 | 处理 |
|---|---|
| `APP_KEY` 缺失 | 执行 `php artisan key:generate` |
| SQLite 文件不存在 | `.env` 仍是默认值，改为 MySQL 后执行 `php artisan config:clear` |
| `Table 'taskhub.sessions' doesn't exist` | 检查 `.env`，必须设置 `SESSION_DRIVER=file`；不要创建 Session Migration |
| Laravel 仍然查询旧配置 | 执行 `php artisan config:clear && php artisan cache:clear && php artisan view:clear` 后重启 `composer run dev` |
| Vite 端口冲突 | 关闭占用 5173 的进程或让 Vite 自动换端口 |
| MySQL Access denied | 检查 `.env` 的用户名和密码 |
| `/tasks` 页面提示 `task` 表不存在 | 尚未执行 `database/schema.sql`，按第 04 章初始化数据库 |
| SSO 配置为空 | 当前正常，Phase 1 尚未拿到总部协议 |
