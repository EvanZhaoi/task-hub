# 12-发布任务模态框

## 本章目标

本章开始第一个正式写入业务：在任务大厅中通过模态框发布招标任务。

本章不创建单独的 `/tasks/create` 页面。发布任务作为任务大厅里的高频操作，直接使用模态框承载。

## 学习目标

- React 如何用 shadcn/Radix Dialog 做模态框。
- Inertia `useForm()` 如何提交 Laravel Web 表单。
- Laravel FormRequest 如何做后端校验。
- Eloquent 如何在一个事务中写入多张表。
- 为什么附件只记录外部 `attachment_id`。
- 为什么任务发布要同时写 `task_event`。

## 最终效果

访问：

```text
http://127.0.0.1:8000/tasks
```

点击右上角“发布任务”，填写表单并提交后：

- 新增一条 `task`。
- 任务状态为 `OPEN`。
- 分配方式为 `BIDDING`。
- 可选附件 ID 写入 `attachment_ref`。
- 写入 `TASK_CREATED` 和 `TASK_PUBLISHED` 两条 `task_event`。
- 页面返回任务大厅并显示“任务已发布。”

## 每一步操作

### 1. 新增 FormRequest

文件：

```text
app/Http/Requests/StoreTaskRequest.php
```

关键点：

- `authorize()` 返回 `true`，因为登录校验由 `EnsureSsoAuthenticated` 负责。
- 字段名使用前端 camelCase，例如 `expectedDelivery`、`biddingDeadline`。
- 后端保存时再映射成数据库字段 `expected_delivery`、`bidding_deadline`。
- `attachmentIds` 是字符串，允许换行、逗号、空格分隔多个外部附件 ID。

### 2. 新增 SnowflakeId 服务

文件：

```text
app/Services/SnowflakeId.php
```

为什么需要它：

- schema.sql 中业务主键是 `BIGINT UNSIGNED`。
- Model 都设置了 `$incrementing = false`。
- 当前阶段不使用 MySQL 自增。
- 暂未接入公司统一 ID 服务。

当前实现是本地轻量方案，后续如果公司提供统一雪花 ID 服务，只替换这个服务。

当前 ID 结构：

```text
相对毫秒 * 1000000 + 随机 worker 前缀 * 1000 + 请求内序号
```

说明：

- 相对毫秒从 2026-01-01 开始计算，避免数字过长。
- 随机 worker 前缀用于降低并发请求同一毫秒生成相同 ID 的概率。
- 请求内序号用于同一个请求里连续创建 `task`、`task_event`、`attachment_ref`。
- 这不是最终分布式 ID 中心；如果公司已有统一 ID 服务，后续应替换本服务。

### 3. 新增 POST /tasks

文件：

```text
routes/web.php
```

新增：

```php
// 发布任务在任务大厅模态框内提交，不单独创建 /tasks/create 页面。
Route::post('/tasks', [TaskController::class, 'store'])->name('tasks.store');
```

该路由放在 `EnsureSsoAuthenticated` 中间件组里，保证只有登录用户能发布任务。

### 4. TaskController 增加 store()

文件：

```text
app/Http/Controllers/TaskController.php
```

事务中完成：

1. 读取当前 SSO 用户。
2. 生成 task ID。
3. 创建 `task`，状态直接为 `OPEN`。
4. 写入发布者 JSON 快照。
5. 写入付款账号最小 JSON 快照。
6. 写入 `TASK_CREATED`。
7. 写入 `TASK_PUBLISHED`。
8. 如果有附件 ID，写入 `attachment_ref`。

注意：

- 当前只做招标发布，`assignment_type = BIDDING`。
- DIRECT 指派后续单独开发，不提前塞进本章。
- 附件文件本体不进 TaskHub，只保存外部附件 ID。

### 5. Layout 增加 actions

文件：

```text
resources/js/Layouts/AppLayout.tsx
```

`AppLayout` 增加 `actions?: ReactNode`，用于放页面级按钮，例如“发布任务”。

这样做比在每个页面自己重写标题栏更稳：

- 标题层级一致。
- 页面右上角操作区一致。
- 后续任务详情、交付、变更也可以复用。

### 6. 增加 Dialog 和 Textarea

文件：

```text
resources/js/components/ui/dialog.tsx
resources/js/components/ui/textarea.tsx
```

Dialog 使用 Radix：

- `Dialog` 控制打开状态。
- `DialogTrigger` 触发打开。
- `DialogContent` 渲染弹窗。
- `DialogClose` 关闭弹窗。

Textarea 用于任务描述和附件 ID 多行输入。

### 7. Index.tsx 增加发布任务表单

文件：

```text
resources/js/Pages/Tasks/Index.tsx
```

关键点：

- 使用 `useForm<TaskCreateForm>()` 管理表单数据、错误和提交状态。
- 表单字段名必须和 `StoreTaskRequest` 一致。
- 提交到 `POST /tasks`。
- 成功后关闭弹窗、清空表单。
- Laravel redirect 回任务大厅，Inertia 自动刷新列表。

表单字段：

| 前端字段 | 后端校验字段 | 数据库字段 |
|---|---|---|
| `title` | `title` | `task.title` |
| `description` | `description` | `task.description` |
| `budget` | `budget` | `task.budget` |
| `expectedDelivery` | `expectedDelivery` | `task.expected_delivery` |
| `biddingDeadline` | `biddingDeadline` | `task.bidding_deadline` |
| `complexity` | `complexity` | `task.complexity` |
| `paymentAccountId` | `paymentAccountId` | `task.payment_account_id` |
| `paymentAccountName` | `paymentAccountName` | `task.payment_account_snapshot.accountName` |
| `attachmentIds` | `attachmentIds` | `attachment_ref.attachment_id` |

### 8. Flash 成功提示

后端：

```php
return redirect()
    ->route('tasks.index')
    ->with('success', '任务已发布。');
```

Inertia 中间件共享：

```php
'flash' => [
    // 表单提交成功后，Controller 可以通过 redirect()->with('success', ...) 给页面一次性提示。
    'success' => $request->session()->get('success'),
],
```

前端读取：

```tsx
const { flash } = usePage<SharedPageProps>().props;
```

## 常见错误

| 错误 | 原因 | 处理方式 |
|---|---|---|
| 419 CSRF 错误 | 表单没有走 Inertia 或 CSRF token 缺失 | 使用 Inertia `useForm().post()`，不要手写 fetch |
| `id` 不能为空 | 忘记生成 BIGINT 主键 | 确认使用 `SnowflakeId->next()` |
| 发布时间保存失败 | `biddingDeadline` 字段名和 Request 不一致 | 前端字段必须叫 `biddingDeadline` |
| 附件重复错误 | 同一个附件 ID 重复关联 | 后端 `attachmentIds()` 已去重 |
| 弹窗提交后不关闭 | 没有在 `onSuccess` 中关闭 | `setIsCreateOpen(false)` |
| 页面没有成功提示 | `flash` 没共享给 Inertia | 检查 `HandleInertiaRequests` |

## 本章总结

本章完成了 TaskHub 第一个写入业务闭环：

```text
React Dialog 表单
↓
Inertia useForm
↓
Laravel FormRequest
↓
TaskController 事务
↓
task / attachment_ref / task_event
↓
返回任务大厅
```
