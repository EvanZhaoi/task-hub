# 08-最小业务闭环

## 1. 本章目标

本章开始做一个非常小的业务闭环，但仍然不开发完整 TaskHub 业务。

目标是通过一个只读任务列表，让你理解：

```text
MySQL
↓
Laravel Model 查询
↓
Controller 整理数据
↓
Inertia::render()
↓
React Page 接收 props
↓
浏览器展示
```

本章只读取 `task` 表，不做发布任务、投标、选标、交付、权限或表单提交。

## 2. 学习目标

完成本章后，你应该理解：

- 为什么需要 Controller。
- Controller 和 Route 分别负责什么。
- Eloquent Model 如何查询数据库。
- 为什么 Controller 不直接返回 HTML。
- Inertia props 如何把 PHP 数据传给 React。
- React 页面如何用 TypeScript 描述后端传来的数据结构。
- 空数据时为什么也要设计页面状态。

## 3. 最终效果

访问：

```text
http://127.0.0.1:8000/tasks
```

你会看到一个任务列表页面：

- 如果数据库有任务，会展示最多 10 条任务。
- 如果数据库为空，会展示空状态。
- 页面只读，不提供新增、修改、删除、投标等动作。

## 7. 每一步操作

### 7.1 修改路由

文件：`routes/web.php`

完整代码：

```php
<?php

use Inertia\Inertia;
use App\Http\Controllers\TaskController;
use Illuminate\Support\Facades\Route;

// 首页仍然是 Phase 1 应用外壳验证页。
Route::get('/', function () {
    return Inertia::render('Welcome', [
        'appName' => config('app.name'),
    ]);
});

// 任务列表页。
// 当浏览器访问 /tasks 时，Laravel 会执行 TaskController@index。
Route::get('/tasks', [TaskController::class, 'index'])->name('tasks.index');
```

为什么这样做：

- `routes/web.php` 只负责“URL 进来后交给谁处理”。
- 首页逻辑很少，可以继续保留闭包。
- `/tasks` 已经要查数据库并整理数据，所以交给 Controller。

与 Spring Boot 对照：

```java
@GetMapping("/tasks")
public String index(Model model) {
    return "tasks/index";
}
```

不同点是：Laravel 这里不是返回 Blade 模板名，而是后面由 Controller 返回 `Inertia::render('Tasks/Index')`。

验证：

```bash
php artisan route:list
```

应该看到：

```text
GET|HEAD  tasks  tasks.index
```

### 7.2 创建 Controller

文件：`app/Http/Controllers/TaskController.php`

完整代码：

```php
<?php

namespace App\Http\Controllers;

use App\Models\Task;
use Illuminate\Support\Str;
use Inertia\Inertia;
use Inertia\Response;

class TaskController extends Controller
{
    public function index(): Response
    {
        // 使用 Eloquent 查询 task 表。
        // select 明确列出页面需要的字段，避免把整行数据无意识传给前端。
        $tasks = Task::query()
            ->select([
                'id',
                'title',
                'description',
                'budget',
                'final_amount',
                'expected_delivery',
                'final_delivery',
                'bidding_deadline',
                'status',
                'assignment_type',
                'complexity',
                'created_by',
                'created_by_snapshot',
                'payment_account_snapshot',
                'created_at',
            ])
            ->latest('created_at')
            ->limit(10)
            ->get()
            ->map(function (Task $task): array {
                // JSON 快照字段允许为空。
                // 先转成空数组，后面读取 displayName 等字段时就不会出现 null 下标警告。
                $createdBySnapshot = $task->created_by_snapshot ?? [];
                $paymentAccountSnapshot = $task->payment_account_snapshot ?? [];

                return [
                    // 前端只负责展示，所以这里把 ID 转成字符串，避免 JS 大整数精度问题。
                    'id' => (string) $task->id,
                    'title' => $task->title,
                    'description' => Str::limit(strip_tags((string) $task->description), 120),
                    'amountLabel' => $this->amountLabel($task),
                    'expectedDelivery' => $task->expected_delivery?->toDateString(),
                    'finalDelivery' => $task->final_delivery?->toDateString(),
                    'biddingDeadline' => $task->bidding_deadline?->format('Y-m-d H:i'),
                    'status' => $task->status,
                    'assignmentType' => $task->assignment_type,
                    'complexity' => $task->complexity,
                    'createdBy' => $task->created_by,
                    'createdByName' => $createdBySnapshot['displayName'] ?? $task->created_by,
                    'departmentName' => $createdBySnapshot['departmentName'] ?? null,
                    'paymentAccountName' => $paymentAccountSnapshot['accountName'] ?? null,
                ];
            })
            ->values();

        // 返回 React 页面 Tasks/Index。
        // 它实际对应 resources/js/Pages/Tasks/Index.tsx。
        return Inertia::render('Tasks/Index', [
            'tasks' => $tasks,
        ]);
    }

    private function amountLabel(Task $task): string
    {
        // 任务成交后优先展示 final_amount，否则展示预算 budget。
        $amount = $task->final_amount ?? $task->budget;

        return '¥'.number_format((float) $amount, 2);
    }
}
```

为什么 Controller 要整理数据：

- 数据库字段是后端结构，不一定适合前端直接展示。
- 前端需要的是页面 ViewModel，例如 `amountLabel`、`createdByName`。
- Controller 在这里承担“把数据库模型整理成页面 props”的职责。

为什么不建 Repository：

- 这一章只有一个简单查询。
- Eloquent 已经足够表达。
- 过早加 Repository 会让学习重点从请求流程变成分层模板。

### 7.3 理解 Model 查询

本章没有修改 `app/Models/Task.php`，但使用了它。

如果你当前项目中没有这个文件，请先学习第 07 章。最小生成命令是：

```bash
php artisan make:model Task
```

但仅生成空 Model 还不够。因为 TaskHub 的表名是单数 `task`，不是 Laravel 默认推断的复数 `tasks`，所以至少要配置：

```php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Task extends Model
{
    // TaskHub 数据库表名是 task。
    // 如果不写，Laravel 默认会查 tasks 表。
    protected $table = 'task';

    // TaskHub 使用雪花 ID，不使用 MySQL 自增主键。
    public $incrementing = false;

    // 主键值是整数类型。
    protected $keyType = 'int';
}
```

完整字段、类型转换和关联关系请以第 07 章和当前仓库 `app/Models/Task.php` 为准。

关键点：

```php
Task::query()
    ->latest('created_at')
    ->limit(10)
    ->get();
```

含义：

- `Task::query()`：从 `task` 表开始构造查询。
- `latest('created_at')`：按创建时间倒序。
- `limit(10)`：只取 10 条，避免学习页面一次拉太多数据。
- `get()`：执行 SQL，并返回 Eloquent Collection。

与 Spring Boot 对照：

```java
taskRepository.findTop10ByOrderByCreatedAtDesc();
```

区别：

- Laravel Eloquent 直接从 Model 发起查询。
- Spring Boot JPA 通常通过 Repository 查询。

### 7.4 创建 React 页面

文件：`resources/js/Pages/Tasks/Index.tsx`

页面职责：

- 声明 `TaskListItem` 类型，对应 Controller 传来的 props。
- 显示任务列表。
- 数据为空时显示空状态。
- 使用与原型相近的浅色背景、卡片、状态徽章和紧凑信息排布。

关键代码：

```tsx
type TaskListItem = {
    id: string;
    title: string;
    description: string;
    amountLabel: string;
    expectedDelivery: string | null;
    finalDelivery: string | null;
    biddingDeadline: string | null;
    status: 'DRAFT' | 'OPEN' | 'ASSIGNED' | 'COMPLETED' | 'FAILED' | 'CANCELLED';
    assignmentType: 'BIDDING' | 'DIRECT';
    complexity: 'LOW' | 'MEDIUM' | 'HIGH';
    createdBy: string;
    createdByName: string;
    departmentName: string | null;
    paymentAccountName: string | null;
};

type TaskIndexProps = {
    tasks: TaskListItem[];
};

export default function TaskIndex({ tasks }: TaskIndexProps) {
    return (
        <main>
            {tasks.length === 0 ? (
                <div>暂无任务数据</div>
            ) : (
                tasks.map((task) => (
                    <article key={task.id}>{task.title}</article>
                ))
            )}
        </main>
    );
}
```

为什么要写 TypeScript 类型：

- 后端 props 一旦改名，前端能更快发现问题。
- 读代码时能看出页面需要哪些字段。
- Java 开发者可以把它理解成前端 DTO。

这里 `TaskListItem` 对应的是 Controller 中整理后的数组，不是数据库原始字段。比如数据库中是 `created_by`，传给前端后变成 `createdBy`；数据库中是 `budget` / `final_amount`，传给前端后变成更适合展示的 `amountLabel`。

这一层类型定义的作用类似 Java 中给前端返回的 Response DTO：

```java
public record TaskListItemResponse(
    String id,
    String title,
    String amountLabel,
    String status
) {}
```

注意：TypeScript 类型不会在浏览器运行时自动校验数据，它主要用于开发阶段提示和编译检查。真正的服务端数据安全仍然要靠 Laravel 查询、校验和权限控制。

为什么要处理空状态：

- 新数据库很可能没有任务数据。
- 空数据不是错误，页面应该明确告诉开发者“链路已经跑通，只是没有数据”。

### 7.5 Inertia 页面名如何匹配文件

Controller 中：

```php
return Inertia::render('Tasks/Index', [
    'tasks' => $tasks,
]);
```

React 文件：

```text
resources/js/Pages/Tasks/Index.tsx
```

匹配规则来自 `resources/js/app.tsx`：

```tsx
const page = pages[`./Pages/${name}.tsx`];
```

所以：

```text
Tasks/Index
→ ./Pages/Tasks/Index.tsx
→ resources/js/Pages/Tasks/Index.tsx
```

Linux 大小写敏感，因此 `Tasks`、`Index.tsx` 的大小写必须一致。

## 8. 每一步为什么这么做

本章刻意只做“读”：

- 避免过早进入表单、校验、事务、权限。
- 先把 Route、Controller、Model、Inertia、React 的协作链路打通。
- 数据库为空时也能验证页面链路。

代码中的职责划分：

| 文件 | 职责 |
|---|---|
| `routes/web.php` | URL 到 Controller 的映射 |
| `TaskController` | 查询数据并整理 props |
| `Task` Model | 映射 `task` 表并执行查询 |
| `Tasks/Index.tsx` | 展示 props |
| `app.tsx` | 根据 Inertia 页面名找到 React 文件 |

## 10. 常见错误

| 错误 | 常见原因 | 处理 |
|---|---|---|
| `Class "App\Models\Task" not found` | 只完成了 01 和 03，还没有创建 Task Model | 先完成第 07 章，确认 `app/Models/Task.php` 存在 |
| `Page not found: Tasks/Index` | React 文件路径或大小写不一致 | 确认文件是 `resources/js/Pages/Tasks/Index.tsx` |
| `Table 'taskhub.task' doesn't exist` | 没有执行 `database/schema.sql` | 按第 04 章初始化数据库 |
| 页面 500 | 数据库连接或 PHP 异常 | 查看 `storage/logs/laravel.log` |
| `tasks` 在 React 中是 `undefined` | Controller props 名称和 React props 类型不一致 | 确认都是 `tasks` |
| TypeScript 报状态类型错误 | 后端返回值不在前端联合类型中 | 检查数据库枚举和前端 `TaskListItem['status']` |
| 页面样式不像原型 | 没有沿用原型的布局、卡片、状态色 | 对照 `docs/prototype/pages/03-task-hall.html` |

## 11. 本章总结

本章完成了第一个最小业务闭环：

```text
浏览器 /tasks
↓
routes/web.php
↓
TaskController@index
↓
Task::query()
↓
MySQL task 表
↓
Inertia::render('Tasks/Index', props)
↓
resources/js/Pages/Tasks/Index.tsx
↓
浏览器展示任务列表
```

你需要重点记住：

- Route 只做分发。
- Controller 组织页面数据。
- Model 查询数据库。
- Inertia 把 PHP 数据交给 React。
- React 只负责展示 props。
