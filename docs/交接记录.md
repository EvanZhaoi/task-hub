## 2026-06-12 · UI 全面打磨

**操作人**：Harry
**协作方**：Evan（产品负责人）

### 上下文
Evan 反馈"现在的样式并不美观，帮我优化前端UI"。检查发现多处违反 DESIGN.md，主要问题是：emoji 装饰泛滥（§8 明令禁止），页面背景全白（§2.1 应是 #fafafa），且组件实现参差。

### 做了什么
- **纠正 DESIGN.md §8 违规**：全部 emoji（💰📅⏰📌👤🕐✅❌⏳💡🔍✓✕📋📎）→ Lucide 图标
- **页面背景**：`body` 改 `#fafafa`（DESIGN.md §2.1 token）
- **新建 `<UserAvatar>` 组件**：渐变色 + 首字母，按角色配色（紫/橙/黑），4 尺寸变体，复用于详情页投标行、ProfileView、AppHeader
- **精修 6 个视图 + 1 个组件**：
  - `TaskCard`：`Meta` 子组件 inline-flex 带 icon，title hover 变 primary 色
  - `AppHeader`：logo 渐变 + Layers 图标，用户信息两行布局（姓名 + 部门），切换 select 右上角加 ChevronDown
  - `HomeView`：搜索框加 `Search` icon，状态筛选改为 card 内胶囊按钮组，复杂度假名 chip 风格统一，空状态带 Inbox 图标
  - `TaskDetailView`：完整重排，标题旁加大尺寸 complexity 徽章 + 状态在右上角，操作按钮全部带 icon
  - `CreateTaskView`：两种模式卡 hover 阴影 + 微动效（border + 位移），表单字段加 hint 文本，提交按钮加 Sparkles
  - `ProfileView`：顶部用户卡 + 三栏统计（发布/投标/接单），任务行加 ChevronRight
  - `BossGanttView`：图例统一 `<Legend>` 组件，tip 改 Info icon
- **动效与交互动效**：
  - 页面进入：`.page-enter / fadeUp 200ms`
  - 卡片 hover：边框 + 阴影平滑过渡（150ms）
  - 输入框 focus：border + ring 颜色变化
  - 按钮 hover：微动效（如创建任务卡 hover translate-y-[-1px]）
- **基础设施**：
  - `Button` 加 `icon-sm` 尺寸变体
  - `globals.css` 加页面背景、淡入动画、输入框 focus 样式

### 关键决策
- **彻底去 emoji**：DESIGN.md 是个遵守依据，不能“以后再说”。这次一次性清干净
- **背景 `#fafafa` vs 全白**：从“死白” → “近黑画布”之间的过渡，Linear 风格
- **不引入外部字体**：DESIGN.md §3 锁定系统字体，不破例
- **不引入 UI 库控件以外的东西**：所有图标走 Lucide，动画走 Tailwind + CSS keyframe
- **不重设计语言**：仅精修，不动 Linear+Vercel 主调

### 上下文
上午 Evan 让我加"难易程度"字段（简单/普通/困难），v0.5 上线。13:10 Evan 反领：把概念从"**难易程度**"重新定义为"**复杂程度**"，值从"简单/普通/困难"改成"**简单 / 中等复杂 / 高度复杂**"。

为避免"字段名 difficulty + 文案 复杂程度"的语义割裂，**同时重命名**：

| 维度 | 上午版（已废弃） | 当前版（v0.5 锁定） |
|---|---|---|
| 概念名 | 难易程度 | **复杂程度** |
| 字段名 | `difficulty` | `complexity` |
| 英文枚举 | `EASY / NORMAL / HARD` | `LOW / MEDIUM / HIGH` |
| 中文标签 | 简单 / 普通 / 困难 | **简单 / 中等复杂 / 高度复杂** |
| 默认值 | NORMAL（普通） | MEDIUM（中等复杂） |
| 组件 | `DifficultyBadge` | `ComplexityBadge` |
| CSS 类 | `.badge-easy/-normal/-hard` | `.badge-low/-medium/-high` |

### 做了什么
- **DB** (`schema.sql`)：列名 `difficulty` → `complexity`，注释 + 索引名同步改
- **类型** (`api/types.ts`)：`Difficulty` → `Complexity`，值改为 `LOW/MEDIUM/HIGH`
- **组件**：删 `DifficultyBadge.tsx`，新建 `ComplexityBadge.tsx`
- **CSS** (`globals.css`)：三个 badge 类重命名
- **视图**：`TaskCard` / `TaskDetailView` / `CreateTaskView` / `HomeView` 全部重命名引用 + 中文文案
- **状态** (`stores/tasks.ts`)：`filterDifficulty` / `setDifficultyFilter` → `filterComplexity` / `setComplexityFilter`
- **mock 数据**：8 条任务改为新枚举值（CRM 屎山=HIGH、订单导出=LOW 等）
- **文档**：
  - 需求文档 v0.5：§6.1 字段描述、§13 L16、§11 MVP 必须做、版本历史都更新
  - 模块与数据库 v0.7：4.2.1 task 表、4.4 索引策略、版本历史都更新
  - `DESIGN.md` §2.3：表头 + 行名 + 值都改
  - 本交接记录段落直接覆盖上午那段

### 关键决策
- **字段名跟概念名一致**：避免后端开发者读 `difficulty` 字段但代码注释/UI 都写"复杂度"的认知错位
- **枚举 LOW/MEDIUM/HIGH 替代 EASY/NORMAL/HARD**：跟"复杂程度"语义贴合（"低复杂度"比"简单"更准确）
- **默认值 = MEDIUM**：跟"未标 = 中等"的直觉对齐
- **不影响定价**：MVP 只作标签 + 筛选，二期再考虑接定价/接单意愿排序
- **历史数据**：还未真上线，mock 全部同步改；真上线时如已有数据需一次性 UPDATE
