**操作人**：Harry
**协作方**：Evan（产品负责人）

### 做了什么
- 加了 **任务难度**字段：`简单 EASY / 普通 NORMAL / 困难 HARD`
- 写入 `task` 表：`difficulty TINYINT NOT NULL DEFAULT 1` + 复合索引 `(difficulty, status)`
- 前端：
  - 类型：`frontend/src/api/types.ts` 加 `Difficulty` 联合类型 + `Task` / `CreateTaskRequest` 字段
  - 组件：新建 `DifficultyBadge.tsx`（绿/灰/红），样式加进 `globals.css`（`.badge-easy` / `.badge-normal` / `.badge-hard`）
  - 表单：`CreateTaskView.tsx` 加三按钮切换器（EASY/NORMAL/HARD），必填，默认 NORMAL
  - 显示：`TaskCard.tsx` 标题旁 + `TaskDetailView.tsx` 顶部 + 信息卡都加徽章
  - 筛选：`HomeView.tsx` 加"难度"筛选行（全部 + 三个 chip），状态：加进 `stores/tasks.ts`
- 文档：
  - `需求文档.md` v0.4 → v0.5：§6.1 Task 加字段、§13 遗留 L9 备注 + 新 L16 已解决、§11 MVP 必须做加一行、版本历史新增
  - `模块与数据库设计.md` v0.6 → v0.7：4.2.1 task 表加字段 + 索引、4.4 索引策略表加一行、版本历史新增
  - `database/schema.sql`：task 表 DDL 加 difficulty 列 + 索引
  - `DESIGN.md` §2.3 新增"难度徽章"颜色规范

### 关键决策
- **默认值**：普通（NORMAL），跟"难度未知 = 中等"的直觉对齐
- **不影响定价**：MVP 只用作标签 + 筛选，二期再考虑接定价/接单意愿排序
- **必填**：发布表单必填，避免历史脏数据
- **筛选交互**：跟状态筛选分开一行，用 badge 色 chip，不挤在一排
- **不动老板视图**：老板视图默认按付款账号筛，不加难度筛（避免过多筛选器）

### 历史数据迁移
- 已有 8 条 mock 任务按业务场景填了不同难度（CRM 屎山=HARD、订单导出=EASY 等）做演示
- 真上线时如果 v0.6 之前已有数据，需一次性 UPDATE 设默认值 1（NORMAL）

### 进度里程碑
- 2026-06-12：v0.5 需求 + v0.7 模块设计 + schema.sql + 前端全套同步升级

- 平台只做"任务流转 + 完成确认"，资金由公司内部系统管

- 管理员**不进系统**（先线下确认金额）

- 老板视图：所有任务甘特图 + 颜色按状态分 + 按付款账号筛选 v0.4 新增

- 任务详情页：描述为主体（不再像副标题）
