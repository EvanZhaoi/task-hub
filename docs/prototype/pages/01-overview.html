系统总览

状态机 · 角色矩阵 · 数据模型 · 通知规则。一页看懂 TaskHub MVP。

1. 三类角色（边界平等）

P
任务发布者

任何员工

  • 浏览任务大厅
  • 发布任务（招标）
  • 直接指名
  • 投标（自己不能投自己发的）
  • 选标、确认完成
  • 取消自己的任务
D
接单开发者

任何员工

  • 浏览任务大厅
  • 投标（单人 / 多人协作）
  • 协商延期
  • 申请变更（金额/需求/交期）
  • 交付 / 撤回投标
  • 同样可以发布任务
B
老板

拥有全局视图权限

  • 看所有任务的甘特图视图
  • 按付款账号筛选
  • 按状态 / 时间范围筛选
  • 不能直接操作任务（去线下）
  • 老板加塞属于发布任务

3. 操作权限矩阵（按当前用户角色 × 任务状态）

操作 发布者 中标者 其他用户 备注
浏览任务（OPEN/历史） ✓ ✓ ✓ 全员公开
投标 ✗ ✓ ✓ 不能投自己发的
选标 ✓ — — 发布者专属
直接指名 ✓ — — 发布者专属
协商延期 ✓ ✓ — 双方
变更字段 ✓ ✓ — 双方确认
取消任务 ✓ ✓ — DRAFT/OPEN 单方；ASSIGNED 双方
确认完成 ✓ — — 发布者专属
查看他人投标 ✓ ✗ ✗ 互相不可见（防串标）
老板视图 — — 仅老板 隐藏入口

4. 数据模型（4 张核心表）

task 任务主表
id · title · description (HTML)
budget · final_amount
expected_delivery · final_delivery
bidding_deadline · status
assignment_type · version
primary_assignee_id · assigned_bid_id
created_by_snapshot · created_at
bid 投标表
id · task_id · revision_no
amount · delivery_date
proposal · status · active_key
member → bid_member
created_at · updated_at
UNIQ(active_key)
—— 保留撤回历史
attachment_ref 附件引用表
id · owner_type · owner_id
attachment_id
uploaded_by · created_at
owner: TASK/BID/DELIVERY/
CHANGE_REQUEST
—— 外部文件服务
task_event 任务事件表
id · task_id
event_type · operator_id
from_status · to_status
related_type · related_id
event_data (JSON) · remark
created_at
—— 时间线 + 审计

不存：付款账号（外部引用）/ 用户表（外部 + Redis 缓存）/ 系统配置（二期）。

5. 通知规则（MVP 简化版）

事件 通知对象 通道
收到投标 发布者 站内 邮件
被选中 中标者 站内 邮件
未中 （中标后自然知，不主动） —
状态变更 双方 站内
流标 发布者 + 原投标者 站内
完成 双方 站内 邮件
协商延期发起 对方 站内

企微 / 钉钉集成：二期。
