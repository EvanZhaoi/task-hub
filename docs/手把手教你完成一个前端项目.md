## 第 0 章：项目介绍

### TaskHub 是什么？

一个**公司内部任务交易平台**：任何员工可以发布开发任务，开发者可以投标，发布者选标，任务进入进行中，验收完成。

**它解决了什么问题**：
- 公司有大量开发任务，但开发资源**分散在各个部门**
- 已有**虚拟货币制度**（≈ 真实奖金）
- 缺乏市场化的资源调配机制

**功能范围**（MVP）：
- 任务发布（招标 + 直接指名）
- 投标 + 选标
- 文件附件
- 任务进行中的延期、取消、完成
- 老板全局视图（甘特图）
- 个人时间线
- 任务搜索 + 分页
- 6 个用户身份（3 种角色）

### 我们要做的

不是空 demo，是**真实可演示**的项目：
- 8 个任务、6 个用户、11 个投标的**完整 mock 数据**
- 5 个 View 全部功能（不只是占位）
- 6 个 shadcn 组件实际用上
- 用户切换器（demo 不同角色）

### 1.1 我们选了哪些技术？

| 层 | 选择 | 版本 | 为什么 |
|---|---|---|---|

| 路由 | **React Router 7** | 7.x | 数据路由 + lazy |
| 状态 | **Zustand 5** | 5.0 | 比 Redux 简单 10 倍 |
| 样式 | **Tailwind v4** | 4.3 | CSS-first，类名即样式 |
| 组件库 | **shadcn/ui** | latest | MIT 商用，复制粘贴 |
| 底层 | **Radix UI** | 1.1 | 无样式可访问性原语 |

| 图标 | lucide-react | 0.469 | 现代、tree-shake 友好 |
| 日期 | dayjs | 1.11 | 比 moment 小 97% |

### 1.2 为什么 React？

| 备选 | 优点 | 缺点 | 结论 |
|---|---|---|---|
| **React** | 生态最大，招聘友好 | 学习曲线稍陡 | ✅ |
| Vue | 中文友好，简单 | 生态比 React 小 | 备选 |
| Svelte | 性能好 | 生态小，企业用得少 | 不选 |

> **建议**：如果你 Java 基础好且想转岗前端，**React 是最优选择**——岗位最多。

### 1.3 为什么 Tailwind v4（不写 CSS）？

**传统方式**：每个组件写一个 `.css` 文件
- 痛点 1：命名冲突（`.title` 跟别人的撞了）
- 痛点 2：删组件忘了删 CSS
- 痛点 3：改个颜色要改 50 个文件

**Tailwind 方式**：类名即样式
```tsx
<h1 className="text-2xl font-semibold text-foreground">标题</h1>
```
- 颜色 / 间距 / 字号 都是**类名**
- 改设计 token，全站跟着变
- 删除组件，CSS 一起删（编译器 tree-shake 掉）

**v3 → v4 的关键变化**：v4 用 **CSS-first 配置**，不再需要 `tailwind.config.js`：
```css
/* v3: tailwind.config.js 里配 theme */
/* v4: globals.css 里写 @theme 块 */
@theme inline {
  --color-primary: var(--primary);
}
```

### 1.4 为什么 shadcn/ui（不是 antd）？

| 备选 | 优点 | 缺点 |
|---|---|---|
| **shadcn/ui** | 复制粘贴，代码归你，零 bundle 锁定 | 组件需要自己加 |
| antd / element-plus | 组件全 | 风格难定制，体积大，锁版本 |
| 自己写 | 完全自由 | 1-2 周工作量 |

**shadcn 的核心思想**：**它不是 npm 依赖，是复制粘贴模式**。组件代码复制到你的 `src/components/ui/`，是你的代码，想改就改。

> 💡 看到 `import { Button } from '@/components/ui/button'` 时要知道——这个 Button 就是你工程里的代码，不是从 npm 装的。

### 1.5 为什么 Zustand（不是 Redux）？

```ts
// Redux：3 个文件 + 大量 boilerplate
// Zustand：1 个文件，10 行搞定

import { create } from 'zustand'
const useStore = create((set) => ({
  count: 0,
  inc: () => set((s) => ({ count: s.count + 1 })),
}))
```

**对比**：
- Redux：action / reducer / dispatch / store / provider，5 个概念
- Zustand：state + actions，2 个概念

对于 MVP，**Zustand 完胜**。

## 第 3 章：样式系统——Tailwind v4

> Tailwind v4 是 2024 年的大版本变化。**最大变化**：CSS-first 配置，不再需要 `tailwind.config.js`。

### 3.1 配置 PostCSS（v4 新方式）

**v3** 的 `postcss.config.js`：
```js
export default { plugins: { tailwindcss: {}, autoprefixer: {} } }
```

**v4** 的 `postcss.config.js`：
```js
export default { plugins: { '@tailwindcss/postcss': {} } }
```

**差异**：
- 插件改名（`tailwindcss` → `@tailwindcss/postcss`）
- **不再需要 `autoprefixer`**（v4 自带浏览器前缀）

### 3.2 globals.css 写主题

**v3** 用 `tailwind.config.js` 配主题：
```js
// tailwind.config.js (v3)
module.exports = { theme: { extend: { colors: { primary: '#5e6ad2' } } } }
```

**v4** 用 CSS `@theme` 块：
```css
/* src/styles/globals.css */
@import "tailwindcss";

:root {
  /* shadcn 约定：HSL 值，不用 rgb() */
  --primary: 240 56% 65%;       /* 你的品牌色 #5e6ad2 */
  --background: 0 0% 100%;
  /* ... 20 个变量 ... */
}

@theme inline {
  --color-primary: var(--primary);  /* 把 :root 变量映射成 Tailwind tokens */
  --color-background: var(--background);
  /* ... 对应每个变量 ... */
}

@layer base {
  * { @apply border-border; }  /* Tailwind 帮你生成 border-border 类 */
  body { @apply bg-background text-foreground; }
}
```

**关键模式（shadcn 标准）**：
1. `:root` 里定义 HSL 值（不是 hex，是 "240 56% 65%" 这种）
2. `@theme inline` 里把 `:root` 变量映射成 `--color-*` tokens
3. Tailwind 自动生成 `bg-primary` / `text-foreground` 等类
4. 想改主题色？只改 `:root` 里一行，全站跟着变

### 3.3 状态徽章（项目自定义）

```css
@layer components {
  .badge {
    @apply inline-block px-2 py-0.5 rounded-full text-xs font-medium whitespace-nowrap;
  }
  .badge-open      { @apply bg-blue-100 text-blue-800; }
  .badge-assigned  { @apply bg-orange-100 text-orange-800; }
  .badge-completed { @apply bg-green-100 text-green-800; }
  .badge-failed    { @apply bg-gray-200 text-gray-700; }
}
```

> 💡 `@layer components` 让你定义的类**优先级**高于 Tailwind 默认类，且**会被 tree-shake** 掉没用的。

### 3.4 在 main.tsx 引入

```tsx
// src/main.tsx
import './styles/globals.css'  // 一定要在所有 import 之前
```

## 第 4 章：组件库——shadcn/ui

> shadcn/ui 是**复制粘贴模式**的组件库。**不是 npm 依赖**。

### 4.1 cn() 工具

shadcn 所有组件都依赖 `cn()` 合并 className：

```ts
// src/lib/utils.ts
import { clsx, type ClassValue } from 'clsx'
import { twMerge } from 'tailwind-merge'

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

**为什么需要它**：
- `clsx` 处理条件 className（`isActive && 'bg-blue-500'`）
- `twMerge` **去重 + 解决冲突**（后者覆盖前者）
- 没有它，`'p-2 p-4'` 会同时应用两个 padding，行为不确定

### 4.2 Button 组件

```tsx
// src/components/ui/button.tsx
import * as React from 'react'
import { Slot } from '@radix-ui/react-slot'
import { cva, type VariantProps } from 'class-variance-authority'
import { cn } from '@/lib/utils'

// 1. 用 cva 定义变体
const buttonVariants = cva(
  // 基础类（所有变体共享）
  'inline-flex items-center justify-center gap-2 rounded-md text-sm font-medium transition-colors focus-visible:outline-none focus-visible:ring-2 disabled:pointer-events-none disabled:opacity-50',
  {
    variants: {
      variant: {
        default: 'bg-primary text-primary-foreground hover:bg-primary/90',
        destructive: 'bg-destructive text-destructive-foreground hover:bg-destructive/90',
        outline: 'border border-input bg-background hover:bg-accent',
        ghost: 'hover:bg-accent hover:text-accent-foreground',
        link: 'text-primary underline-offset-4 hover:underline',
      },
      size: {
        default: 'h-10 px-4 py-2',
        sm: 'h-9 px-3 text-sm',
        lg: 'h-11 px-8',
        icon: 'h-10 w-10',
      },
    },
    defaultVariants: { variant: 'default', size: 'default' },
  },
)

// 2. Props 类型：cva 的变体 + 原生 button 属性
export interface ButtonProps
  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<typeof buttonVariants> {
  asChild?: boolean
}

// 3. forwardRef + Slot 模式（让父组件能传任意元素作为 Button）
const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variant, size, asChild = false, ...props }, ref) => {
    const Comp = asChild ? Slot : 'button'
    return <Comp className={cn(buttonVariants({ variant, size, className }))} ref={ref} {...props} />
  },
)
Button.displayName = 'Button'

export { Button, buttonVariants }
```

**关键模式**：
1. **cva 定义变体**：variant (外观) + size (大小)
2. **forwardRef 转发 ref**：父组件能拿到底层 DOM 节点的引用
3. **Slot 模式**：`asChild` 让 `<Button asChild><Link to="...">...</Link></Button>` 能渲染成 `<a>` 而不是 `<button>`

### 4.3 Dialog 组件（最复杂）

```tsx
// src/components/ui/dialog.tsx
import * as DialogPrimitive from '@radix-ui/react-dialog'
import { X } from 'lucide-react'
import { cn } from '@/lib/utils'

const Dialog = DialogPrimitive.Root
const DialogTrigger = DialogPrimitive.Trigger
const DialogPortal = DialogPrimitive.Portal
// ... 其他原语导出

const DialogOverlay = React.forwardRef<...>(({ className, ...props }, ref) => (
  <DialogPrimitive.Overlay
    ref={ref}
    className={cn('fixed inset-0 z-50 bg-black/40 data-[state=open]:animate-in ...', className)}
    {...props}
  />
))

const DialogContent = React.forwardRef<...>(({ className, children, ...props }, ref) => (
  <DialogPortal>
    <DialogOverlay />
    <DialogPrimitive.Content
      ref={ref}
      className={cn('fixed left-1/2 top-1/2 z-50 ... -translate-x-1/2 -translate-y-1/2 ...', className)}
      {...props}
    >
      {children}
      <DialogPrimitive.Close className="absolute right-4 top-4 ...">
        <X className="h-4 w-4" />
      </DialogPrimitive.Close>
    </DialogPrimitive.Content>
  </DialogPortal>
))
```

**为什么 Dialog 用 Radix 而不是自己写**：
- 焦点管理（按 Tab 不会跑出弹窗）
- 键盘支持（ESC 关闭）
- Portal 渲染到 body 末尾（不被父级 overflow 影响）
- ARIA 属性（屏幕阅读器友好）

**自己写这些 = 2-3 天工作量 + 大量 bug**。Radix 免费给你。

### 5.3 Axios 客户端（先备好，暂不接）

```ts
// src/api/client.ts
import axios, { type AxiosInstance } from 'axios'

const baseURL = import.meta.env.VITE_API_BASE || '/api'

export const api: AxiosInstance = axios.create({
  baseURL,
  timeout: 15000,
  headers: { 'Content-Type': 'application/json' },
})

// 请求拦截：注入 token
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('taskhub.token')
  if (token && config.headers) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 响应拦截：解包 + 错误处理
api.interceptors.response.use(
  (response) => {
    const body = response.data
    if (body && typeof body === 'object' && 'code' in body) {
      if (body.code === 0 || body.code === 200) return body.data
      return Promise.reject(new Error(body.message))
    }
    return body
  },
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('taskhub.token')
    }
    return Promise.reject(new Error(error.response?.data?.message))
  },
)
```

**后端 ready 后的切换**：store 里把 `MOCK_TASKS` 替换成 `await api.get('/tasks')` 即可，**其他代码不用动**。

## 第 6 章：状态管理——Zustand 5

> Zustand 解决了 React 状态共享问题。**比 Redux 简单 10 倍**。

### 6.1 为什么需要状态管理？

```tsx
// ❌ 不用状态管理：层层传 props
<GrandParent user={user} setUser={setUser}>
  <Parent user={user}>
    <Child user={user} />

// ✅ 用 Zustand：任何地方都能读/写
function Child() {
  const user = useUserStore((s) => s.currentUser)
}
```

### 6.2 第一个 store

```ts
// src/stores/user.ts
import { create } from 'zustand'
import { persist } from 'zustand/middleware'  // localStorage 持久化
import type { User, UserRole } from '@/api/types'
import { MOCK_USERS } from '@/mocks'

interface UserState {
  currentUser: User | null
  isBoss: () => boolean
  isDeveloper: () => boolean
  setUser: (user: User | null) => void
  logout: () => void
}

export const useUserStore = create<UserState>()(  // 注意双括号：curried 形式
  persist(
    (set, get) => ({
      currentUser: MOCK_USERS[0],  // 默认是李雷

      isBoss: () => get().currentUser?.role === 'boss',
      isDeveloper: () => get().currentUser?.role === 'developer',

      setUser: (user) => set({ currentUser: user }),

      logout: () => {
        set({ currentUser: null })
        localStorage.removeItem('taskhub.token')
      },
    }),
    { name: 'taskhub-user' },  // localStorage key
  ),
)
```

**关键概念**：
- `create<T>()` **curried 形式**——v5 必须这样写才能传类型
- `persist` 中间件——自动同步到 localStorage
- `get()` 在 store 内部访问当前 state（实现 `isBoss` 这种派生 getter）

### 6.3 怎么用 store？

```tsx
import { useUserStore } from '@/stores/user'

// 1. 读 state（自动 re-render）
function Header() {
  const currentUser = useUserStore((s) => s.currentUser)
  return <div>你好，{currentUser?.name}</div>
}

// 2. 调 action
function ProfileButton() {
  const setUser = useUserStore((s) => s.setUser)
  return <button onClick={() => setUser(MOCK_USERS[0])}>切到李雷</button>
}

// 3. 派生计算（推荐用 Zustand 里的 getter，避免每次 render 重算）
const isBoss = useUserStore((s) => s.isBoss())  // 注意双括号：是函数调用
```

### 6.4 第二个 store：tasks

```ts
// src/stores/tasks.ts
interface TasksState {
  list: Task[]
  filterStatus: TaskStatus | 'all'
  searchQuery: string
  page: number
  pageSize: number

  setFilter: (status: TaskStatus | 'all') => void
  setSearch: (q: string) => void
  setPage: (p: number) => void
  getById: (id: string) => Task | undefined
  setStatus: (id: string, status: TaskStatus) => void
}

export const useTasksStore = create<TasksState>()((set, get) => ({
  list: [...MOCK_TASKS],

  filterStatus: 'all',
  searchQuery: '',
  page: 1,
  pageSize: 4,

  setFilter: (status) => set({ filterStatus: status, page: 1 }),  // 重置到第 1 页
  setSearch: (q) => set({ searchQuery: q, page: 1 }),
  setPage: (p) => set({ page: p }),

  getById: (id) => get().list.find((t) => t.id === id),

  setStatus: (id, status) =>
    set((s) => ({
      list: s.list.map((t) => (t.id === id ? { ...t, status, updatedAt: new Date().toISOString().slice(0, 10) } : t)),
    })),
}))
```

**store 设计原则**：
- **每个业务域一个 store**（user / tasks / bids / attachments / changelogs）
- **state + actions 一起放**，不分开文件
- **getter 也放里面**（避免组件每次 render 算一遍）

### 6.5 ⚠️ 坑 3：v5 的 create 签名

```ts
// ❌ 旧版写法（v4）
export const useStore = create<State>((set) => ({...}))

// ✅ v5 必须 curried
export const useStore = create<State>()((set) => ({...}))
```

差一个括号！如果 v5 写错，TypeScript 会报"Expected 1-2 arguments, but got 0"——**最坑的是错误信息完全看不懂**。

## 第 7 章：路由——React Router 7

> React Router 7 引入了**数据路由**模式（`createBrowserRouter`），与之前的 `<BrowserRouter>` 不同。

### 7.1 路由配置

```tsx
// src/router/index.tsx
import { createBrowserRouter, Navigate, Outlet } from 'react-router'
import { AppHeader } from '@/components/AppHeader'

const routes = [
  {
    path: '/',
    element: <Layout />,
    children: [
      { index: true, lazy: () => import('@/views/HomeView').then((m) => ({ Component: m.HomeView })) },
      { path: 'task/:id', lazy: () => import('@/views/TaskDetailView').then((m) => ({ Component: m.TaskDetailView })) },
      { path: 'create', lazy: () => import('@/views/CreateTaskView').then((m) => ({ Component: m.CreateTaskView })) },
      { path: 'profile', lazy: () => import('@/views/ProfileView').then((m) => ({ Component: m.ProfileView })) },
      { path: 'boss', lazy: () => import('@/views/BossGanttView').then((m) => ({ Component: m.BossGanttView })) },
      { path: '*', element: <Navigate to="/" replace /> },
    ],
  },
]

function Layout() {
  return (
    <div className="min-h-screen bg-background">
      <AppHeader />
      <main>
        <Outlet />  {/* React Router 6+ 用 Outlet 替代了 router-view */}
      </main>
    </div>
  )
}

export const router = createBrowserRouter(routes)
```

**关键点**：
- `lazy: () => import(...).then(m => ({ Component: m.X }))` —— 路由级 code splitting
- `<Outlet />` 渲染子路由（不是 `<router-view />`）
- `path: '*'` 兜底路由，未匹配的跳首页

### 7.2 入口


**关键点**：
- `RouterProvider` 是 v7 的方式（不是 `<BrowserRouter><App /></BrowserRouter>` 包裹）

### 8.4 GanttChart（自写 CSS，避大依赖）

```tsx
// src/components/GanttChart.tsx
import { useMemo } from 'react'
import { Link } from 'react-router'

export function GanttChart({ tasks, mode = 'boss' }: { tasks: Task[]; mode?: 'personal' | 'boss' }) {
  // 1. 计算时间范围
  const { start, end } = useMemo(() => {
    if (tasks.length === 0) return { start: new Date(), end: new Date() }
    const dates = tasks.flatMap(t => [
      new Date(t.createdAt).getTime(),
      new Date(t.finalDelivery || t.expectedDelivery || t.createdAt).getTime(),
    ])
    const minD = new Date(Math.min(...dates))
    const maxD = new Date(Math.max(...dates))
    minD.setDate(minD.getDate() - (mode === 'personal' ? 3 : 7))
    maxD.setDate(maxD.getDate() + (mode === 'personal' ? 7 : 14))
    return { start: minD, end: maxD }
  }, [tasks, mode])

  const total = end.getTime() - start.getTime()

  // 2. 渲染每个任务条
  return (
    <div className="rounded-lg border bg-card overflow-x-auto">
      <div className="relative h-6 border-b border-border" style={{ minWidth: 700 }}>
        {/* 月份标尺 */}
      </div>
      <div className="space-y-1.5 p-3" style={{ minWidth: 700 }}>
        {tasks.map(task => {
          const tStart = new Date(task.createdAt).getTime()
          const tEnd = new Date(task.finalDelivery || task.expectedDelivery || task.createdAt).getTime()
          const left = Math.max(0, ((tStart - start.getTime()) / total) * 100)
          const width = Math.max(2, ((tEnd - tStart) / total) * 100)
          return (
            <div key={task.id} className="flex items-center gap-3 h-8">
              <Link to={`/task/${task.id}`} className="w-48 text-sm truncate">{task.title}</Link>
              <div className="flex-1 relative h-8 bg-muted/30 rounded">
                <div
                  className={`absolute h-6 rounded ${STATUS_BAR_CLASS[task.status]} ...`}
                  style={{ left: `${left}%`, width: `${width}%`, top: 4 }}
                >
                  <span className="truncate">{task.title}</span>
                </div>
              </div>
            </div>
          )
        })}
      </div>
    </div>
  )
}
```

**核心算法**：每条任务的位置 = `(taskStart - viewStart) / (viewEnd - viewStart) * 100%`。

### 9.1 HomeView（任务大厅 + 搜索 + 筛选 + 分页）

**核心模式**：用 `useMemo` 在客户端做筛选 + 搜索 + 分页。

```tsx
export function HomeView() {
  const list = useTasksStore((s) => s.list)
  const filter = useTasksStore((s) => s.filterStatus)
  const search = useTasksStore((s) => s.searchQuery)
  const page = useTasksStore((s) => s.page)
  const setPage = useTasksStore((s) => s.setPage)

  const filtered = useMemo(() => {
    let arr = [...list]
    if (filter !== 'all') {
      if (filter === 'FAILED') arr = arr.filter(t => t.status === 'FAILED' || t.status === 'CANCELLED')
      else arr = arr.filter(t => t.status === filter)
    }
    const q = search.trim().toLowerCase()
    if (q) {
      arr = arr.filter(t => t.title.toLowerCase().includes(q) || (t.description || '').toLowerCase().includes(q))
    }
    return arr.sort((a, b) => b.createdAt.localeCompare(a.createdAt))
  }, [list, filter, search])

  const totalPages = Math.max(1, Math.ceil(filtered.length / pageSize))
  const safePage = Math.min(page, totalPages)
  const start = (safePage - 1) * pageSize
  const pageItems = filtered.slice(start, start + pageSize)

  return (
    <div>
      {/* 搜索框 + 筛选按钮 */}
      {/* 任务列表 */}
      {/* 分页（仅当 > pageSize 时显示） */}
    </div>
  )
}
```

**为什么用 useMemo**：
- 筛选 + 搜索 + 排序是 O(n)，每次 render 都跑浪费
- `useMemo([list, filter, search])` 只在依赖变化时重算

## 第 12 章：常见问题 & 进阶话题

### 12.1 富文本编辑器（暂未集成）

我们的 mock 数据用了**手写 HTML**。生产应该用富文本编辑器：

| 库 | 协议 | 推荐 |
|---|---|---|

| Quill | MIT | 也可 |
| Lexical | MIT | Meta 出品，较新 |

集成思路：

```tsx
import StarterKit from '@tiptap/starter-kit'

const editor = useEditor({

```

### 12.2 真实后端对接

**目前**：store 直接用 mock 数据。

**后端 ready 后**：
1. 在 `src/api/*.ts` 加 `api.get(...)` 调用
2. store 里加 `fetch` action：
   ```ts
   async fetchList() {
     const tasks = await taskApi.listTasks()
     set({ list: tasks })
   }
   ```
3. View 里 `useEffect(() => store.fetchList(), [])`

**不要做的事**：
- ❌ 在 View 里直接调 API（破坏 store 的统一性）
- ❌ 混用 mock 和 API（数据源要统一）

### 12.3 错误处理

MVP 阶段 `alert()` 就够了。生产应该用：
- 顶部 toast 通知（react-hot-toast / sonner）
- 错误边界（`componentDidCatch`）
- 401 自动跳登录

### 12.4 性能优化

| 优化点 | 怎么做 | 何时做 |
|---|---|---|
| 列表虚拟化 | `react-window` | 列表 > 100 条 |
| 路由 lazy | `lazy: () => import(...)` | 路由 > 5 个 |
| 组件 memo | `React.memo` | 组件 render 慢 |
| Zustand 选择器 | `useStore((s) => s.x)` | 避免全 state 订阅 |
| 图片懒加载 | `loading="lazy"` | 图片 > 10 张 |

### 12.5 测试

| 类型 | 工具 | 写什么 |
|---|---|---|
| 单元测试 | Vitest | store actions、utils 函数 |
| 组件测试 | Vitest + Testing Library | Button/Card 行为 |
| E2E | Playwright | 完整流程（发布→投标→选标） |

**推荐**：先写**业务关键路径**的 E2E（用 Playwright），其他靠手动测。覆盖率 80% 是过度优化。
