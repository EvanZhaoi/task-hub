# TaskHub 原型

> ⚠️ **这是原型，不是最终代码**。使用 mock 数据，仅用于演示流程和确认 UI。

## 🎯 怎么跑

**最简单：直接双击 `index.html`**

Mac / Windows 都可以。浏览器会打开原型页面，所有功能立即可用。

> 💡 推荐用 Chrome / Edge / Safari 打开

## 🎬 演示剧本

### 场景 1：任务大厅（默认）
打开 `index.html` → 看到所有任务列表，可按状态筛选

### 场景 2：完整流程（推荐演示顺序）

**第一步：发布招标任务**
1. 顶部下拉切换到 `陈PM · 发布者`
2. 导航点「发布任务」→ 选「招标模式」
3. 填个标题 → 发布

**第二步：开发者投标**
1. 顶部下拉切换到 `李雷 · 开发者`
2. 进任务详情 → 点「💰 投标」
3. 填金额 / 交期 / 方案 → 提交

**第三步：选标**
1. 切回 `陈PM`
2. 进你刚发的任务 → 看到投标列表 → 点「选 TA」
3. 任务变「进行中」

**第四步：完成 / 延期 / 取消**
1. 点「✓ 确认完成」→ 任务完结
2. 或切到 `韩梅梅` 演示「申请延期」（需要切回原任务发布者确认，模拟演示可省）
3. 切到发布者演示「取消任务」

**第五步：老板视图**
1. 切到 `Evan · 老板`
2. 导航点「老板视图」
3. 看甘特图

**第六步：个人中心甘特图**
1. 切到任意开发者身份（如 `韩梅梅`）
2. 点「我的」→ 「我接的」Tab
3. **看到下方有"我的任务时间线"甘特图**（仅显示你接的任务）

**第七步：富文本编辑**
1. 切到 `陈PM` → 「发布任务」→ 招标模式
2. 在"详细描述"里看到 Quill 富文本编辑器（有加粗、斜体、列表、代码、链接等按钮）
3. 写完发布，再看任务详情 → 描述以富文本形式渲染

### 场景 3：直接指名（跳过招标）
1. 切到 `张总 · 发布者`
2. 「发布任务」→ 选「直接指名」
3. 选被指名人（如李雷）→ 发布
4. 任务**直接**进入进行中

### 场景 4：流标 + 复制新建
1. 进任务 `t6 内部论坛功能（流标）`
2. 看到「流标」状态 + 「📋 复制新建」按钮
3. 点一下 → 任务被复制为 DRAFT 草稿，可编辑

### 场景 5：老板加塞 / 协商延期
1. 进任务 `t7 老板加塞CEO演示数据准备`
2. 看到「变更记录」里有「延期 2 天」的记录
3. 显示有变更历史

## 🛠 技术栈

- Vanilla HTML + Alpine.js 3
- Tailwind CSS (Play CDN)
- **Quill 1.3.7** (富文本编辑器，snow 主题，CDN)
- 纯静态，无后端

**设计参考**：[DESIGN.md](../DESIGN.md)（Linear + Vercel 风格）

## ⚠️ 局限性（原型阶段）

- ❌ 无登录/认证
- ❌ 数据不会持久化（刷新会重置）
- ❌ 文件上传是占位符
- ❌ 没有站内消息通知（只显示一个 toast）
- ❌ 任务超时自动流标 不实现
- ❌ 多人协作投标的"协作成员"字段 暂不实现 UI
- ❌ **任务描述含富文本但不防 XSS**（生产环境需用 DOMPurify 过滤）
